# 화면 연출 및 애니메이션 명세 (Presentation & Animation)

> **[문서 목적]** 본 명세서는 게임 내 화면 전환, 버튼 인터랙션 등 모든 시각적/청각적 연출 지침을 정의하는 핵심 기획 명세서입니다.

## 0. 기획 의도 (Design Intent)
> 시청각적 화면 연출의 핵심 기획 의도는 플레이어의 모든 조작에 대한 즉각적인 피드백과 서사적 **[타격감(Impact)]**을 극대화하는 데 있습니다.
> 확신을 가지고 판정을 내리는 순간(EJECT 추방 셔터 하강, 사이렌 등) 강력한 시각·청각적 밀도를 제공해 긴장감과 쾌감을 동시에 주며, UI 마이크로 애니메이션(버튼 호버/클릭 바운스)을 통해 조작의 물리적 질감을 살리고 몰입을 유지하도록 돕습니다.

## 1. 트랜지션 및 글로벌 연출
게임 내 화면 전환 및 시스템 상태 변경 시 공통으로 적용되는 효과.

- **화면 전환 (Fade In/Out)**:
  - 블랙 아웃(Black-out) 형태의 페이드 효과 적용. (Fade-out 0.5초 / Fade-in 0.5초)
- **오디오 연출 (BGM / SFX)**:
  - BGM 전환 시 1.0초 Crossfade (기존 트랙 1.0초 Fade-out, 신규 트랙 1.0초 Fade-in).
  - **라디오 플레이리스트 셔플 (Radio Randomization)**: 게임 시작 시점(혹은 씬 진입 시) BGM 플레이리스트가 Fisher-Yates 셔플 등 무작위로 재배치되어 매번 다른 순서로 음악이 재생됨.
  - 단기 반복 SFX(도장 타격음, 키보드 타자음 등)는 Pitch를 `0.9 ~ 1.1` 사이에서 랜덤하게 적용하여 기계감을 상쇄.
- **모달 창 플로팅 UI**:
  - 등장 시 Background Dim 60% 페이드인, 모달 스케일 `0.95 -> 1.0` 확대 (Ease-Out, 0.2초).
  - 퇴장 시 반대 (Ease-In, 0.15초).

## 2. 캐릭터 감정 (Stress State) 연출 규칙
NPC의 Stress 지수에 따라 Idle 모션과 텍스트 출력 방식이 동적으로 변경됩니다.

| 연출 상황 | 시각적 모션 (Visual Motion) | 대화 텍스트 연출 (Text Output) |
| :--- | :--- | :--- |
| **기본 대기 (Idle)** | Y축 기준 `+5f` 높이로 `1.5초` 주기의 미세한 둥실거림 (Ease-InOut-Sine) | 0.05초 간격으로 일정하게 타이핑 출력 |
| **대화 반응 (Talk Reaction)** | Y축으로 `+15f` 높이를 `0.12초` 만에 가볍게 한 번 바운스 (OutQuad -> InQuad) | 일반 폰트 및 고정 속도 유지 |
| **표정(감정) 변화 시** | 스프라이트 교체 시 X축 기준 `±10f` 폭으로 `0.05초` 간격의 빠른 진동 (흠칫 연출) | 감정 상태 표출 유지 |

---

## 3. 핵심 기믹 (Gimmick) 타격감 연출

### 3.1. PASS (통과) 도장 연출
- **시각**: 도장 오브젝트가 스케일 `3.0 -> 1.0`으로 강하게 내려찍힘 (Ease-In-Expo 모션, `0.15초`). 이후 1.0초간 유지 후 사라짐.
- **피드백**: 서류가 닿는 순간 타격음과 함께 대상 UI 패널(서류)에 진동(DOShakePosition) 강도 `20f`로 `0.2초`간 발생.

### 3.2. EJECT (추방) 격벽(Bulkhead) 연출
- **시각**: 셔터가 `0.2초` 만에 강하게 닫히며(Ease-In-Quad), `0.5초` 동안 대기(Hold) 후 `0.8초`에 걸쳐 천천히(Ease-Out-Sine) 원래 위치로 복귀.
- **피드백**: 버튼 조작 즉시 닫히는 SFX와 경보음(Alarm) SFX가 동시 발생.

## 4. 인트로, 엔딩 및 씬 전환(Transition) 연출

### 4.1. 일일 날짜(Day) 전환 연출 (`VFXManager_Scene.cs`)
매일 아침(새로운 Day 시작) 화면이 밝아질 때의 UI 전환 연출.
- **Fade**: 검은 화면(Alpha 1f)에서 `1.0초` 동안 서서히 투명(Alpha 0f)해지며 화면이 밝아짐.
- **텍스트 등장**: "Day X" 텍스트가 `0.5배`에서 `1.0배`로 `0.5초`간 통통 튀며(OutBack) 등장.
- **텍스트 퇴장**: `2.0초` 대기 후 다시 `0.5배`로 쪼그라들며(InBack) `0.5초`간 페이드아웃 됨.

### 4.2. 인트로 씬 시퀀스
- 흑백 일러스트와 화면 노이즈 필터, 타자음 중심의 로우파이 분위기 컷신.
- 대사가 타이핑된 후 스킵 방지를 위해 2.0초 대기 후 다음 컷으로 자동 전환.
- 컷신 종료 후 1.5초 Fade-out 되며 메인 데스크 씬으로 즉시 이행.

### 4.3. 엔딩 분기 연출
조건 충족 후 5일 차 정산 종료 시 화면이 암전(0.5초)되며 4가지 엔딩 연출 중 하나로 진입.
1. **Game Over (파산)**: 우울한 하모니카 사운드, 쫓겨난 스켈레톤의 뒷골목 컷신 (BGM 단절).
2. **Normal (이달의 사축)**: 웅장한 팡파르 후 한숨과 야근 서류 더미 컷신 (음향 반전 효과).
3. **Bad (카르텔)**: 불타는 골드와 폭력적인 상황을 상징하는 베이스 BGM 및 타격음 (화면 진동).
4. **True (영앤리치)**: 통쾌한 재즈 BGM과 함께 승리감 넘치는 사직서 투척 컷신 (경쾌한 사운드 믹스).

## 5. 전역 UI 인터랙션 (Global UI Interaction)

프로젝트 전체 UI에서 공통으로 적용되는 시스템 기반의 마이크로 애니메이션(`VFXManager_UI.cs` 바인딩) 명세.

### 5.1. 버튼 인터랙션 (Button Hover & Click)
- **Hover Enter (마우스 진입)**: 오브젝트 스케일이 0.2초에 걸쳐 `1.1배`로 확대 (Ease-Out-Back 모션으로 약간의 반동 동반).
- **Hover Exit (마우스 이탈)**: 0.2초에 걸쳐 `1.0배`로 원상 복구 (Ease-Out-Sine 모션으로 부드럽게 복귀).
- **Pointer Down (클릭)**: 스케일이 0.1초 동안 `0.95배`로 축소되었다가 원래대로 돌아오는 Yoyo 루프 연출 (총 2회 반복되며 쫀득한 클릭감 부여).

### 5.2. 모달 팝업 연출 (Popup Open / Close)
튜토리얼, 환경설정, 자판기 등 각종 모달 창이 호출/해제될 때 일관된 스케일+페이드 효과를 제공함.
- **Open (팝업 호출)**: 
  - 딤(Dim) 배경 Alpha 값이 0.4초 만에 `0.0 -> 1.0`으로 Fade-in 됨.
  - 모달 패널 스케일이 `0.5 -> 1.0`으로 0.4초 동안 Ease-Out-Back(Overshoot: 1.7)으로 통통 튀며 등장.
- **Close (팝업 닫기)**:
  - 딤(Dim) 배경 Alpha 값이 0.3초 만에 `1.0 -> 0.0`으로 Fade-out 됨 (동시에 레이캐스트 차단).
  - 모달 패널 스케일이 `1.0 -> 0.5`로 0.3초 동안 Ease-In-Back으로 안쪽으로 빨려 들어가며 퇴장.
