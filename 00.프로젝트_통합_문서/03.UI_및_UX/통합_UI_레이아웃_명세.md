# 핵심 UI 레이아웃 및 UX 명세 (Core UI/UX Layout Specification)

> **[문서 목적]** 본 명세서는 게임 내 모든 UI 레이아웃과 상호작용 규칙에 대한 단일 진실 공급원(SSOT)으로 작성된 최종 기획 명세서입니다. 모든 배치값과 앵커, 상호작용 로직은 실제 Unity 구현 상태를 기반으로 합니다.

## 0. 기획 의도 (Design Intent)
> 본 게임의 UI 레이아웃은 유저의 '탐색 피로도'를 최소화하기 위해 철저하게 규격화되었습니다. 
> 신분증, 일일 메모, 매뉴얼 등의 정보를 **[카드형 인터페이스]**로 규격화 및 표준화하여, 플레이어가 화면 내 여러 정보를 산발적으로 쫓지 않고 직관적으로 **[읽기 ➔ 비교 ➔ 확신]**으로 이어지는 자연스러운 시선 흐름을 경험하게 유도하는 것이 핵심 기획 의도입니다.

## 1. 글로벌 UI 규칙 (Global UI Rules)
게임 전반에 공통으로 적용되는 디자인 및 상호작용 규격.

| 항목 | 규격 및 내용 | 비고 |
| :--- | :--- | :--- |
| **해상도 (Resolution)** | 1920 × 1080 (16:9) 기준 | `Canvas_MainUI` Scale Mode: Expand |
| **기본 폰트 (Font)** | 나눔고딕 ExtraBold (40px 기준) | 시스템 UI 텍스트용 |
| **특수 폰트 (Special)**| 수필(Handwriting) 또는 타자기 폰트 | 메모지, 서류 등 아날로그 에셋 |
| **상호작용 (Hover)** | 마우스 hover 시 Scale 1.1배 확대 및 외곽선 강조 | 모든 상호작용 가능한 오브젝트 |
| **디자인 톤앤매너** | 탁한 로우파이(Lo-Fi) 아날로그 톤 | 베이지, 브라운관 초록, 낡은 붉은색 |

## 3. UI 및 데이터 연동 아키텍처
* `Canvas_MainUI` 하위에 배치되며, Unity의 `UIManager` 컴포넌트(`GameUIController`, `ChatUIManager`, `IDCardDisplay`, `ShopUIManager` 등)가 각 패널을 바인딩하여 런타임 제어.
* 사원증 텍스트 및 상점 인벤토리 상태 등은 `DataManager`에서 로드된 ScriptableObject 및 SessionData를 동적으로 주입하여 화면을 렌더링.
