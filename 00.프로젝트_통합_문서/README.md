# 프로젝트 통합 문서 (Notion 이관용)

이 폴더(`00.프로젝트_통합_문서`)는 기존 `Legacy` 폴더의 기획 및 개발 일지 양식을 계승하며, **실제 구현된 Unity 프로젝트(MCP 연결)를 기반으로 작성되는 최종 통폐합 문서**를 보관합니다. 최종적으로 Notion 등에 이관하기 위한 목적으로 사용됩니다.

## 🎯 최종 목적 (Ultimate Goal)
1. **레거시 문서의 통폐합**: 기존에 작성되었던 모든 마크다운(md) 기획서 및 산출물은 `Legacy` 폴더로 이관하고, 오직 **실제 유니티에 구현된 기능과 데이터**만을 기반으로 새로운 통합 문서를 작성합니다.
2. **Notion 이관 준비**: 새롭게 작성되는 이 폴더 안의 문서들은 최종적으로 Notion에 깔끔하게 이관 및 정리하기 위한 마스터본(Master Copy) 역할을 합니다.
3. **AI 에이전트(Antigravity) 작업 지침서**: 이 문서는 AI 에이전트가 재시작되었을 때, 프로젝트의 방향성과 원칙을 즉시 파악하고 작업을 재개하기 위한 기준점입니다.

## 📌 절대 원칙 (Strict Rules)
1. **단방향 동기화 (읽기 전용)**: AI는 MCP를 통해 유니티 프로젝트의 구조(하이어라키, 스크립트, 프로퍼티 등)를 **읽기(Read-only)만** 수행합니다. 어떠한 경우에도 유니티 프로젝트 측의 에셋이나 코드를 수정해서는 안 됩니다.
2. **문서 수정 범위**: 파일 추가, 수정, 삭제 등의 모든 쓰기(Write) 작업은 오로지 현재의 `Document_Hero_Exposed` 레포지토리 내의 `00.프로젝트_통합_문서` 하위에서만 이루어집니다.
3. **단일 진실 공급원 (SSOT)**: Unity 프로젝트의 실제 코드가 곧 기획입니다. 기존 Legacy 기획서는 오직 문서의 '형식(양식)'과 '과거 맥락'을 참고하기 위한 용도로만 쓰이며, 내용은 반드시 유니티 실제 구현체와 1:1로 일치해야 합니다.
4. **템플릿 기반 작성**: 모든 산출물은 사전에 정의된 하위 폴더의 템플릿 양식을 준수하여 작성됩니다.

## 🚀 현재 작업 진행 상황 (Status & Next Steps)
- **완료**: 기존 문서를 Legacy로 이관 완료. 새로운 최상위 폴더(`00.프로젝트_통합_문서`) 및 양식 세팅 완료.
- **완료**: Unity MCP 연결을 통해 전체 하이어라키, 70개 스크립트, 5개 씬을 전수 조사하여 구조 파악 보고서 업데이트 완료.
- **완료**: 모든 핵심 명세서에 `0. 기획 의도 (Design Intent)` 섹션 추가 완료.
- **완료**: `06.오디오_및_VFX`, `07.월드_및_오브젝트` 폴더 채우기 완료. MainScene/SettlementScene UI 명세 추가 완료.
- **상태**: 최종 빌드 상태. 더 이상의 기능 추가 없음. 포트폴리오/Notion 이관 준비 완료.
