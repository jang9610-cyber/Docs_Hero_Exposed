# 경제 시스템 및 상점 기획 (Economic System)

> **[문서 목적]** 본 명세서는 게임 내 경제 시스템과 재화 정산 밸런스를 정의하는 핵심 기획 명세서입니다.

## 0. 기획 의도 (Design Intent)
> 상점 보급 및 정산 시스템은 게임의 **[핵심 루프(심문-대조-판정-정산-보급)]**를 완결짓는 필수 단계입니다.
> 유저로 하여금 뇌물 수수(부정부패)와 철저한 심사(보안 유지) 사이에서 도덕적/전략적 딜레마를 느끼게 하며, 구매한 보급 아이템을 통해 다음 회차의 심문을 유리하게 이끄는 '선순환 구조'를 만드는 것이 주된 의도입니다. 궁극적으로 다회차 플레이 시 '완벽한 줄타기(영앤리치)'라는 도전적인 목표를 부여합니다.

## 1. 개요
게임 내 경제 시스템은 '사내 복지 상점(자판기)'을 통해 정산 재화를 소모하고 아이템을 활용하여 심문을 유리하게 이끄는 전략적 요소를 제공합니다. 또한 재화의 누적 및 뇌물 수수는 5일 차 엔딩 분기의 핵심 기준이 됩니다.

## 2. 경제 밸런스 규칙 (Economic Balance)

### 2.1 재화 획득 및 지출 (5일 기준)
매일 5명의 NPC가 방문하는 5일 차 루프(총 25명)를 기준으로 재화가 정산됩니다.

| 항목 | 금액 (G) | 발생 조건 및 상세 설명 |
| :--- | :--- | :--- |
| **기본 심문 성공** | **+100** | 정확한 판정(PASS/EJECT)으로 NPC 심문 완료 시 지급 |
| **오판 (실수)** | **0** | 판정 실패 시 수당 미지급 |
| **뇌물 수수** | **+200** | 뇌물 수락 후 PASS 시 획득 (수당 미지급, 뇌물 스택 +1, 보안 평점 0점) |
| **일일 고정 지출** | **-250** | 매일 정산 시 '사내 복지 기금 및 방세' 명목으로 강제 차감 |
| **감사관 통과 벌금** | **-400** | 위장한 감사관(Inspector)의 함정에 넘어가 PASS 시 다음 날 정산에서 차감 |

### 2.2 최종 엔딩 분기 조건
5일 차 종료 시, 누적된 '보안 평점'과 '뇌물 스택'을 대조하여 엔딩을 결정합니다.

| 엔딩 명칭 | 발동 조건 | 설명 |
| :--- | :--- | :--- |
| **👑 영앤리치 (True)** | 평점 18↑ AND 뇌물 4↑ | **"완벽한 줄타기"** - 보안 유지와 비자금 챙기기 동시 달성 |
| **💀 카르텔 (Bad)** | 평점 17↓ AND 뇌물 4↑ | **"보안 붕괴"** - 무분별한 뇌물 수수로 본사 점령당함 |
| **👔 우수 사축 (Normal)** | 평점 18↑ AND 뇌물 3↓ | **"가난한 충견"** - 보안은 지켰으나 돈이 없어 회사에 예속됨 |
| **🗑️ 해고 / 파산 (GameOver)**| 평점 17↓ OR 잔액 0↓ | **"가차 없는 토사구팽"** - 능력 부족 또는 벌금 누적으로 파산 |

## 3. 아이템 명세 (Items)
아이템의 구체적 기능은 `ItemManager`에서 관리되며 `NPCChatManager` 등과 상호작용합니다.

| 아이템명 | 가격 | 구매 제한 | 핵심 효과 (Effect) | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **고농축 에너지 드링크** | **150G** | 슬롯당 1개 | 사용 즉시 대상 NPC **심문 턴(대화 기회) +2** | 일회성 컨텍스트 증가 |
| **최고급 연초** | **200G** | 슬롯당 1개 | 사용 즉시 대상 NPC **스트레스 수치 +50** | 감정 수치 조작용 |
| **불법 최면 어플** | **450G** | 슬롯당 1개 | 사용 후 1회 질문에 대해 **'진실' 답변 강제** | 프롬프트 락(Lock) |

## 4. UI 및 예외 처리

### 4.1 상점 진입 로직
- **진입 트리거**: 일일 정산 화면(바탕화면)에서 `자판기.exe` 아이콘 더블 클릭 시 호출.
- **상점 닫기**: 모달 창 우측 상단 [X] 버튼으로 닫기 (정산 화면 복귀).
- **턴 종료**: `일일정산.exe`에서 `[퇴근]` 클릭 시 다음 일차로 진행 (`SceneManager` 연동).

### 4.2 예외 처리 로직 (Edge Cases)
- **재화 부족**: 클릭 시 구매 불가 SFX 및 시스템 경고 팝업 발생.
- **파산 처리**: 정산 결과 재화가 0 미만일 경우 `SceneManager`를 통해 강제 게임오버 씬 전환.
- **최면 어플 낭비**: 비정상적인 텍스트 입력 시에도 효과는 소모됨 (환불 불가).
- **엔딩 시점 제한**: 5일차 정산 시 상점 진입 차단, 즉시 엔딩 씬으로 전환.
