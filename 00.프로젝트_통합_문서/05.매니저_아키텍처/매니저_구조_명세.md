# 유니티 매니저 아키텍처 (Manager Architecture)

## 0. 기획 의도 (Design Intent)
> 매니저 아키텍처의 설계 의도는 **[관심사의 분리(Decoupling)와 협업 효율성 극대화]**입니다. 
> 게임 로직(GameManager), 데이터(DataManager), 시청각 피드백(VFX/AudioManager), 그리고 네트워크 통신(OpenAIClient)을 독립된 매니저로 분리함으로써, 개발 과정에서 기획/아트 파트와 프로그래밍 파트가 병렬로 작업할 수 있도록 시스템의 결합도를 낮추고 유지보수성을 높였습니다.

## 1. 씬 최상단 매니저 (Global / Root Managers)
GameScene의 Root에 배치된 전역적 성격의 매니저 객체들입니다. 주로 게임의 수명 주기 및 전역 데이터, 전환을 담당합니다.

| 게임 오브젝트명 | 주요 컴포넌트 | 기능 요약 |
| :--- | :--- | :--- |
| **GlobalUIManager** | `UIManager` | 프로젝트 전체의 최상위 UI 상태와 모달 팝업 등을 총괄합니다. |
| **DataManager** | `DataManager`, `LoadCharacterData`, `LoadSuperiorLog` | NPC 데이터, 상부 로그, 기획 데이터 등 게임 내 정적/동적 데이터를 로드하고 제공합니다. |
| **SceneManager** | `SceneController`, `LoadingManager`, `CutSceneManager` | 씬 전환 로직, 로딩 처리, 컷신 재생 등을 관리합니다. |
| **GameManager** | `GameManager`, `ItemManager`, `AudioSource` | 핵심 게임 루프(정산, 턴 관리), 아이템 구매 및 사용 로직을 중앙 제어합니다. |
| **AudioManager** | `AudioManager` | BGM, SFX 등 오디오 재생 및 볼륨 컨트롤을 전담합니다. |
| **VFX_Managers** | `VFXManager_NPC`, `VFXManager_Scene`, `VFXManager_UI` | NPC 효과, 화면 이펙트, UI 클릭 효과 등 모든 시각적 피드백 효과를 관리합니다. |

---

## 2. ====System==== 계층 (Local Scene System)
게임 씬 내의 상호작용 시스템과 로컬 매니저들을 포함합니다.

| 게임 오브젝트명 | 하위 오브젝트 | 주요 컴포넌트 | 기능 요약 |
| :--- | :--- | :--- | :--- |
| **Main Camera** | - | `Camera`, `UniversalAdditionalCameraData`, `AspectRatioEnforcer` | 해상도 대응 및 URP 카메라 시스템 |
| **UIManager** | - | `GameUIController`, `ChatUIManager`, `ItemTableUI`, `MemosPanel`, `NPCWindow`, `IDCardDisplay` | 인게임 내 특정 패널들(채팅창, 상점 테이블, 신분증, 메모창 등)의 직접적인 조작을 수행합니다. |
| **System** | - | `OpenAIClient`, `NPCChatManager`, `GameVFXManager` | AI와의 네트워크 통신, 프롬프트 조합(`NPCChatManager`), 씬 내부 이펙트 트리거 관리를 담당합니다. |

---

## 3. ====UI==== 계층 (Canvas System)
실제 렌더링되는 UI 그래픽 컴포넌트들이 배치된 계층입니다.

- **Canvas_MainUI**: 해상도 스케일러가 적용된 기본 Canvas 오브젝트.
- **EventSystem**: UI 터치 및 클릭 입력을 처리하는 Unity Event System.

---

## 4. ====WorldObjects==== 계층 (2D World System)
인게임에서 2D 월드에 배치되는 렌더링 요소들입니다.

- **WO_LowBG**: 배경 렌더링.
- **WO_NPC**: `NPCWorldProxy` 스크립트를 통해 현재 등장한 NPC의 Sprite 렌더링 및 애니메이션 대행.
- **BG_NPC_Spawner**: `BackgroundNPCSpawner` 스크립트로 배경에서 움직이는 행인 NPC 등 환경적 생동감 연출.

---

## 5. 데이터 흐름 요약
1. **사용자 행동**: `Canvas_MainUI`에서 입력 발생 (EventSystem)
2. **UI 처리**: `System/UIManager`의 각 패널 스크립트가 입력 수신 (`ChatUIManager`, `IDCardDisplay` 등)
3. **핵심 로직**: `GameManager` (아이템) 또는 `System` (`NPCChatManager`, `OpenAIClient`) 로 전달
4. **결과 반영**: `DataManager`에 상태 업데이트 후, `UIManager`와 `WO_NPC`를 통해 시각적 피드백 및 `VFX_Managers`, `AudioManager` 트리거.

---

## 6. GameState FSM (유한 상태 머신)

`GameManager.currentGameState`가 보관하는 게임 진행 상태입니다. `Update()`에서 매 프레임 체크되며, 상태에 따라 허용되는 행동이 엄격히 제한됩니다.

1. **Intro** — 씬 진입 직후. NPC 입장 연출 완료 후 버튼 활성화.
2. **Playing** — 핵심 심문 상태. `CheckNPCHere()`가 활성화되어 다음 NPC를 자동으로 불러옴.
3. **Finishing** — PASS/EJECT 판정 후 도장·퇴장 연출 중. 모든 입력 차단.
4. **Outro** — 하루치 NPC 전원 처리 완료. 상관 대화 후 정산 씬으로 전환 대기.

> **공통**: ESC 키를 통한 Pause 팝업은 `Finishing` 상태를 제외한 모든 상태에서 허용됩니다.

| 상태 | `CheckNPCHere()` | PASS/EJECT 버튼 | Esc 팝업 | NPC 입력 |
| :--- | :---: | :---: | :---: | :---: |
| `Intro` | ❌ | ❌ (차단) | ✅ | ❌ |
| `Playing` | ✅ | ✅ | ✅ | ✅ |
| `Finishing` | ❌ | ❌ (차단) | ❌ | ❌ |
| `Outro` | ❌ | ❌ (차단) | ✅ | ❌ |

> **스트레스 분노 퇴장**: `TriggerAngryMovement()` 호출 시 `CalculateTrustPoint(false)` 후 `ReadyNextNPC()` → 자동으로 `Playing` 상태로 복귀.

## 7. DataConector 이벤트 버스 아키텍처

`DataConector.cs`는 씬 내 어디서든 `DataManager`의 직접 참조 없이 데이터를 요청/수정할 수 있게 하는 **이벤트 기반 브릿지 패턴**입니다. 의존성 주입 없이도 UI 스크립트들이 안전하게 데이터에 접근할 수 있습니다.

### 7.1. 등록된 이벤트 목록 (DataManager에서 구독)

| 이벤트명 | 방향 | 연결 메서드 | 용도 |
| :--- | :--- | :--- | :--- |
| `DataConector.GetSessionDataRequested` | 요청 → DataManager | `GetSessionData()` | SessionData 객체 참조 요청 |
| `DataConector.GetStaticDataRequested` | 요청 → DataManager | `GetStaticData()` | StaticData 객체 참조 요청 |
| `DataConector.GetSaveDataRequested` | 요청 → DataManager | `GetSaveData()` | SaveData 객체 참조 요청 |
| `DataConector.HasSaveDataRequested` | 요청 → DataManager | `HasSaveData()` | 세이브 파일 존재 여부 확인 |
| `DataConector.OnResetGameRequested` | 명령 → DataManager | `StartNewGame()` | 데이터 전체 초기화 후 새 게임 |
| `DataConector.OnRemoveSavedataRequested` | 명령 → DataManager | `ClearSaveFile()` | 세이브 파일만 삭제 |
| `IPopupBase.OnResetGameRequested` | 팝업 → DataManager | `StartNewGame()` | 팝업에서 게임 리셋 요청 직통 경로 |
| `DataConector.OnGameStateRequested` | 요청 → GameManager | `GetGameState()` | 현재 GameState 조회 |

### 7.2. 사용 예시
씬 내 UI 스크립트(예: `EndUIManager`)가 `DataConector`를 상속하여, `RequestRemoveSavedata()` 한 줄 호출만으로 `DataManager.ClearSaveFile()`을 간접 실행합니다. `DataManager` 인스턴스를 직접 찾지 않아도 됩니다.
