# 프롬프트 조립 엔진 명세 (Prompt Assembling Engine Specification)

> **[문서 목적]** 본 명세서는 `NPCChatManager.cs`와 `OpenAIClient.cs`의 실제 소스코드를 기반으로, NPC와의 대화 흐름에서 프롬프트가 어떻게 조립되고, AI에게 전송되고, 응답이 처리되는지를 상세하게 기술합니다. 본 문서는 본 프로젝트의 가장 핵심적인 기술 기획 명세입니다.

## 0. 기획 의도 (Design Intent)
> 본 프로젝트의 가장 큰 기술적 도전 과제는 **"AI를 단순한 챗봇이 아닌, 게임 기획 의도를 100% 따르는 인격체로 만드는 것"** 이었습니다.
> 이를 위해 단일 고정 프롬프트 대신, **모듈화된 5+1개의 레이어를 런타임에 동적으로 조립**하는 Prompt Assembling Engine을 자체 구현했습니다.
> 이 구조 덕분에 기획 데이터(`ScriptableObject`) 수치 변경이 코드 수정 없이 즉시 AI 행동에 반영되는 완전한 Data-Driven 아키텍처를 달성했습니다.

---

## 1. 프롬프트 데이터 소스 구조 (ScriptableObjects)

프롬프트는 코드에 하드코딩되지 않고, Unity `ScriptableObject` 에셋 파일로 관리됩니다.

| ScriptableObject 타입 | 개수 | 로드 경로 | 역할 |
| :--- | :--- | :--- | :--- |
| `CommonPrompt` | 1개 | `Resources/commonPromptFilePath` | 세계관, 출력 규칙, 스트레스·지능 파라미터 |
| `NpcTypePrompt` | 다수 | `Resources/npcTypePromptFilePath/` 폴더 | NPC 타입별(진짜/위장/감사관 등) 특수 가드레일 |
| `RacePrompt` | 다수 | `Resources/racePromptFilePath/` 폴더 | 종족별(오크/뱀파이어/다크엘프 등) 진짜/가짜 페르소나 |
| `DailyMemoPrompt` | 5개 | `Resources/Data/Prompt/Memo/XX_Memo` | 일차별 특이사항 (Real/Fake 분기) |

> **로드 방식**: `Awake()`에서 `Resources.Load<T>()` 및 `Resources.LoadAll<T>()`로 전량 메모리에 로드 후 캐싱. 런타임 중 재로드 없음.

## 2. 프롬프트 조립 파이프라인 (`MakePrompt()`)

`NPCChatManager.MakePrompt()`는 NPC 등장 시(`SendPrompt`) 및 플레이어 메시지 전송 시(`SendPlayerMessage`) **매 턴마다 호출**되어 최신 상태의 프롬프트를 재조립합니다.

### 2.1. 조립 순서 (레이어 우선순위)

| 순서 | 변수명 | 생성 메서드 | 역할 |
| :---: | :--- | :--- | :--- |
| 1 | `informationPrompt` | `MakeInformationPrompt()` | 공통 규칙 |
| 2 | `basicInfoPrompt` | `MakeBasicInfoPrompt()` | NPC 기본 정보 |
| 3 | `npcTypeInfoPrompt` | `MakeNpcTypePrompt()` | NPC 타입 지침 |
| 4 | `raceInfoPrompt` | `MakeRaceInfoPrompt()` | 종족 페르소나 |
| 5 | `dailyInfoPrompt` | `MakeDailyInfoPrompt()` | 오늘의 특이사항 |
| 6 | `emotionPrompt` | `MakeEmotionPrompt()` | 현재 감정 상태 |
| Optional | `truthPrompt` | `MakeTruthPrompt()` | 아이템 사용 시만 — 진실 강제 지령 |

위 레이어들을 순서대로 문자열 연결(concatenation)하여 **최종 System Prompt 1개**를 완성합니다.

### 2.2. 각 레이어 상세 명세

#### Layer 1 — 공통 규칙 (`MakeInformationPrompt`)
- **출처**: `CommonPrompt.chatRulePrompt` + `CommonPrompt.responseRulePrompt`
- **내용**: 세계관 설정, 금칙어(AI 순응형 인사 금지), JSON 출력 형식 강제, 메타 명령 거부 원칙 등 **모든 NPC에게 공통으로 적용되는 최상위 법칙**
- **변경 시점**: 변경 없음. 항상 고정.

#### Layer 2 — NPC 기본 정보 (`MakeBasicInfoPrompt`)
- **출처**: `DataManager.sessionData.nowNpcInfo` (`CharacterData`)
- **포함 정보**: `이름`, `종족`, `직책/지위`, `출입 목적`, `현재 스트레스 지수`
- **분기 로직**:
  - `KnowledgeType.MissingBasicInfo` 또는 `KnowledgeType.CompleteFake`인 경우: 기본 정보를 제공하지 않고 "기억이 나지 않음"으로 대체. AI가 세계관 내에서 스스로 정보를 창작하도록 유도.
  - 그 외: 실제 `CharacterData` 수치값으로 채움.

#### Layer 3 — NPC 타입 지침 (`MakeNpcTypePrompt`)
- **출처**: `npcTypePrompts` 리스트에서 `profile.npcType`과 일치하는 것 검색
- **내용**: NPC 타입(예: `Inspector`, `Demon`, `Human`, `FakeHuman`)별 특수 행동 가드레일. "진짜 직원처럼 행동", "감사관처럼 반격" 등.
- **태그 치환**: `{Name}`, `{Race}`, `{Position}`, `{Purpose}`, `{IsFake}` 변수를 `CharacterData` 실값으로 치환 (`ProcessTags()`).

#### Layer 4 — 종족 페르소나 (`MakeRaceInfoPrompt`)
- **출처**: `racePrompts` 리스트에서 `profile.raceType`과 일치하는 것 검색
- **분기 로직**:
  - `profile.isPass == true` (진짜 직원): `racePrompt.npcInfoFormat` 사용 (진짜 종족 특성 및 비밀 보유)
  - `profile.isPass == false` (위장 용사): `racePrompt.fakeNpcInfoFormat` 사용 (종족 연기 지침, 허점 탑재)
- **generalInfoFormat**: 위/가짜 공통으로 먼저 추가되는 해당 종족의 일반적인 세계관 설명.

#### Layer 5 — 일일 특이사항 (`MakeDailyInfoPrompt`)
- **출처**: `Resources.Load<DailyMemoPrompt>($"Data/Prompt/Memo/{todayDate:D2}_Memo")`
- **분기 로직**:
  - `KnowledgeType.PerfectTruth` 또는 `KnowledgeType.MissingBasicInfo`: `realMemoContent` 사용 (사건의 세부 정보 완전 파악)
  - 그 외: `fakeMemoContent` 사용 (키워드만 아는 수준)
- **의도**: 매일 다른 특이사항을 통해 반복적인 심문 패턴을 방지하고, 위장자는 표면적인 정보만 알아 추리 포인트를 형성.

#### Layer 6 — 감정 상태 (`MakeEmotionPrompt`)
- **출처**: `CommonPrompt.intelligencePrompts` (지능 레벨별 행동 지침), `CommonPrompt.stressPrompts` (스트레스 구간별 감정 지침)
- **스트레스 절대값 기준**: 비율이 아닌 `currentEmotion.stressThreshold` 절대값(`0~200`)을 기준으로 구간별 행동 지침 선택.
- **포함 정보**: `지능:{intelligenceText}`, `압박감:{stressText}`

#### [Optional] Truth Layer — 진실 아이템 강제 발동
- **트리거**: `isUseTruthItem == true` (아이템 사용 시 플래그 세팅)
- **내용**: `"\n[TruthItem] On"` 문자열을 프롬프트 최하단에 추가
- **효과**: `CommonPrompt.chatRulePrompt`의 `[Special Item Logic]` 섹션이 이 태그를 감지하여 진실 강제 발화 모드 진입.
- **1회 한정**: 응답 수신 후 `isUseTruthItem = false`로 자동 리셋.

## 3. OpenAI API 통신 흐름 (`OpenAIClient.cs`)

### 3.1. NPC 첫 등장 시 (System Prompt 초기화)

```
NPCChatManager.SendPrompt()
    → openAIClient.ClearChatMessage()          // 이전 대화 기록 초기화
    → openAIClient.SetSystemPrompt(MakePrompt()) 
        → chatMessage[0] = {role:"system", content: 완성된 프롬프트}
        → ProcessAIResponse()                  // AI 첫 응답(인사말 생성)
    → chatUIManager.UpdateChat()               // UI 업데이트
```

### 3.2. 플레이어 메시지 턴마다 (턴 방어 레이어 삽입)

```
NPCChatManager.SendPlayerMessage(message)
    → openAIClient.SetSystemPrompt(MakePrompt())  // 매 턴 프롬프트 재주입 (System 갱신)
    → openAIClient.SendMessage(message)
        ├─ 메시지 래핑: <player_speech>태그 + 강제 알림 삽입
        ├─ MetaCmdPattern.IsMatch(message)?
        │   └─ TRUE: chatMessage에 META_ALERT 추가 (탈옥 시도 방어)
        ├─ chatMessage에 TURN_REMINDER 추가 (규칙 희석 방지)
        └─ ProcessAIResponse()
            └─ SendMessageToAI() → API 전송 (타임아웃 8초, 최대 5회 재시도)
```

### 3.3. AI 응답 처리 (JSON 파싱)

AI는 항상 다음 형식의 JSON만 반환하도록 강제됩니다:
```json
{
    "answer": "NPC의 대사 텍스트",
    "stress_change": 정수값 (양수=스트레스 증가, 음수=감소)
}
```

- `{}` 사이의 내용만 추출하여 `JsonUtility.FromJson<AIResponseData>()` 파싱.
- `responseData.stress_change`를 `currentEmotion.stressThreshold`에 누적.
- `responseData.answer`만 대화창에 표시 및 `chatHistory`에 저장.

## 4. 스트레스 시스템 & 감정 상태 갱신

```
매 응답 수신 시:
    currentEmotion.stressThreshold += response.stress_change
    → Mathf.Clamp(0, MAX_STRESS=200)
    → UpdateEmotionType()
        → commonPrompt.stressPrompts 순회하여 현재 stressThreshold에 해당하는 EmotionType 결정
        → npcWindow.ChangeExpression(currentEmotion.EmotionType)  // UI 표정 변경
        → stressThreshold >= 100 → GameManager.TriggerAngryMovement()  // 분노 연출 트리거
```

| 스트레스 구간 | 감정 타입 | 연출 |
| :--- | :--- | :--- |
| 0 ~ 49 | Normal | 차분한 대응 |
| 50 ~ 99 | Flustered | 동요 및 실수 증가 |
| 100 ~ 200 | Angry | 본성 노출 + 분노 연출(`TriggerAngryMovement`) |

## 5. 탈옥 방어 레이어 (Anti-Jailbreak)

`OpenAIClient.cs`에는 플레이어의 메타 명령(탈옥 시도)을 감지·차단하는 다중 레이어가 구현되어 있습니다:

| 레이어 | 구현 방식 | 동작 |
| :--- | :--- | :--- |
| **Meta Command Guard** | `MetaCmdPattern` Regex 탐지 | "해줘", "연기해", "시스템", "jailbreak" 등 키워드 탐지 시 `META_ALERT` 삽입 |
| **TURN_REMINDER** | 매 턴 system 메시지 재삽입 | JSON 형식·스트레스 구간·메타 명령 거부 규칙 리마인드. 장기 대화에서 system 지침이 희석되는 것 방지. |
| **Player Message Wrapping** | `<player_speech>` 태그 | 플레이어 발화를 명확히 구분하여 AI가 유저 입력을 시스템 지시로 오인하지 못하도록 방지. |
| **System Prompt Re-injection** | 매 턴 `SetSystemPrompt()` 재호출 | `chatMessage[0]`(System 역할)을 최신 상태 프롬프트로 매 턴 갱신. |

## 6. 프롬프트 디버깅 도구

- **`#if UNITY_EDITOR` 블록**: 에디터 환경에서만 완성된 최종 프롬프트를 `Assets/PromptDebugLog.txt` 파일로 저장. Unity 콘솔의 텍스트 잘림 문제를 우회.
- **`LogToFileManager.cs`**: MainScene 전용 로그 파일 저장 유틸리티. 빌드에서도 선택적으로 로그 출력 가능.
