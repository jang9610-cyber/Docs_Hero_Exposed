## 2. 게임 세계관 & 시놉시스

### 2.1. 세계관
마왕이 무력 정복 대신 **적대적 M&A**를 통해 합법적으로 세상을 지배한 세계. 판타지 생물들은 **'마왕물산'** 이라는 대기업에서 일반 직장인으로 근무하며, 평화가 찾아오자 실업자가 된 용사들은 **가짜 신분을 위장하여 기업에 잠입**하려 합니다.

### 2.2. 플레이어의 역할
마왕물산 **1층 보안 데스크의 신입 주임(스켈레톤)**. 출근하는 직원들을 심문하여 진짜 몬스터 직원과 위장한 용사를 가려내고, PASS(통과) 또는 EJECT(추방) 판정을 내립니다.

### 2.3. 핵심 긴장 요소
- **비대칭 정보**: 플레이어에게 주어지는 서류와 NPC가 실제로 아는 정보 사이의 갭을 탐색하는 추리 구조
- **윤리적 딜레마**: 밀수꾼의 뇌물을 받아 돈을 챙길지, 원칙을 지킬지 선택
- **감정 압박**: NPC를 심리적으로 몰아붙여 스트레스를 높이고 '가면이 깨지는 순간'을 유도

---

## 3. 핵심 재미 요소

### 3.1. 자유 채팅 기반 심문
정해진 선택지가 아닌 **플레이어가 직접 채팅을 입력**하여 NPC를 심문합니다.
LLM이 NPC의 페르소나를 유지하며 실시간으로 반응하므로, 유도 질문·압박·함정 등 다양한 심문 전술을 자유롭게 시도할 수 있습니다.

### 3.2. 스트레스 엔진 — 감정 변화 연출
NPC가 압박을 받을 때마다 스트레스 수치가 누적됩니다.
- **Normal(0~49)**: 차분한 대응
- **Flustered(50~99)**: 동요 및 TMI 방어 기제
- **Angry(100~200)**: 본성 노출 + 분노 퇴장 연출

위장한 용사는 극한의 스트레스 상황에서 본성(기사·성직자 등의 말투)이 튀어나오도록 설계되어 있습니다.

### 3.3. 경제·엔딩 딜레마
5일간의 선택이 누적되어 4종의 엔딩으로 분기됩니다. 보안 업무와 뇌물 수수를 동시에 달성하는 '영앤리치' 엔딩은 높은 전략적 난이도를 요구하며, 반복 플레이를 유도합니다.

## 4. 코어 게임 루프

1. **[하루 시작 — GameScene]** 상급자(보안실장) 인트로 대사 표시 (일일 특이사항 힌트 포함)
2. NPC 입장 연출 (WalkIn Animation)
3. 플레이어: 신분증 확인 + 채팅 심문 (자유 입력)
4. 스트레스 변화 → 표정 연출 → (선택) 아이템 사용
5. 판정: PASS 또는 EJECT (도장 연출)
6. 경제 보상 계산 → 다음 NPC 대기 → 5명 완료 시 상급자 아웃트로 대사
7. **[일일 정산 — SettlementScene]** 정산 로그 타이핑 → 자판기(아이템) 쇼핑 → 퇴근
8. 5일차 완료 시: CutScene + EndScene 진입 / 아니면 다음 날 GameScene 재개

## 5. 등장 NPC 시스템

### 5.1. NPC 유형 (IdentityTypes) — 4종

| 유형 | 명칭 | PASS 결과 | EJECT 결과 |
| :--- | :--- | :--- | :--- |
| `T_DEMON` | 진짜 마족 직원 | ✅ 정답 (+100G, +1신뢰) | ❌ 오판 (0G, -1신뢰) |
| `T_HUMAN` | 위장 용사(인간) | ❌ 오판 (0G, -1신뢰) | ✅ 정답 (+100G, +1신뢰) |
| `T_INSPECTOR` | 감사관 (함정) | ❌ 패널티 (+100G, -5신뢰) | ✅ 정답 (+100G, +5신뢰) |
| `T_SMUGGLER` | 밀수꾼 (뇌물 제안) | 뇌물 수락 (+500G, -1신뢰) | ✅ 원칙 (+100G, +1신뢰) |

> **스트레스 분노 퇴장**: 스트레스 ≥ 100 시 NPC가 자발적으로 퇴장. EJECT와 동일한 결과로 처리됨.

### 5.2. 등장 종족 — 6종

| 종족 | 소속 팀 / 직무 | 말투 특징 | 위장 용사의 본성(약점) |
| :--- | :--- | :--- | :--- |
| **오크** | 시설관리팀 현장 용역 | 거친 반말, 현장 은어 | 압박 시 기사 말투 노출 |
| **고블린** | 재무회계팀 자금 경리 | 빠른 신경질적 반말, 수치 강조 | 불쌍한 사연에 적선하려는 본성 |
| **다크엘프** | 전략기획팀 브랜드 마케터 | 차가운 기계적 존댓말 | 타인의 아픔에 힐러 본능 노출 |
| **구울** | 시설관리팀 단순 잡부 | 극도로 느린 단답, 악취 일상화 | 압박 시 인간처럼 말이 빨라짐 |
| **뱀파이어** | 전략사업팀 야간 정보원 | 나른하고 오만한 존댓말 | 햇빛·날씨 화제에 긍정 반응 |
| **리치** | 마력전산팀 시스템 관리자 | 논리적·권위적 딱딱한 말투 | 상세 보안 규정 추궁 시 횡설수설 |

### 5.3. 일일 덱 구성 & 셔플
- 하루 5명 NPC를 `NPCType.json`에 정의된 타입 구성표에 따라 배정
- **Fisher-Yates 알고리즘**으로 등장 순서 완전 무작위화
- 같은 날 동일 종족 중복 방지 (최대 30회 재생성 시도)
- `T_DEMON` 50% 확률로 불완전 지식 상태(지식 결함) 부여 (10칸 천장형 가챠)

### 5.4. 지식 불완전성 시스템 (KnowledgeType)

| 유형 | 적용 대상 | 프롬프트 분기 |
| :--- | :--- | :--- |
| `PerfectTruth` | 진짜 마족 (50%) | 실제 신상 + 실제 일일 메모 |
| `MissingBasicInfo` | 진짜 마족 (25%) | 기본 정보 창작 모드 + 실제 일일 메모 |
| `FakeDailyInfo` | 진짜 마족 (25%) | 실제 신상 + 가짜 일일 메모 |
| `CompleteFake` | 인간 위장 (100%) | 기본 정보 창작 + 가짜 일일 메모 |

## 6. 심문 & 스트레스 시스템

### 6.1. 대화 구조
- 플레이어 채팅 입력 → `NPCChatManager` 프롬프트 재조립 → `OpenAIClient` API 전송 → JSON 응답 파싱
- AI 응답 형식: `{ "answer": "대사", "stress_change": 정수 }`
- 스트레스는 0~200 범위로 클램핑 (200은 방어 상한선, 실질 분노 임계값: 100)

### 6.2. 지능(IntelligenceLevel) 별 행동 차이

| 지능 구간 | 특성 | 대화 방식 |
| :--- | :--- | :--- |
| **Low (낮음)** | 구울 계열 | 단답, 감정(짜증·배고픔) 위주, 논리 거부 |
| **Mid (중간)** | 오크·뱀파이어 등 | 공지 단순 전달, 직무 범위 내 성실 응대 |
| **High (높음)** | 고블린·다크엘프·리치 | 정보 재조합, 역공 질문, 규정·전문용어 인용 |

### 6.3. 일일 특이사항 (일차별 힌트 & 심문 포인트)

| 일차 | 특이사항 | 플레이어 판단 포인트 |
| :--- | :--- | :--- |
| Day 1 | 3층 식당 식중독으로 폐쇄 | 식당 화제에 대한 생리적 반응 여부 |
| Day 2 | 엘리베이터 점검 → 계단 이용 강제 | 계단 사용에 따른 신체 피로 반응 |
| Day 3 | 사무실 영하 20도 | 추위에 대한 신체 감각 반응 여부 |
| Day 4 | 마왕님 생신 → 월급 10% 강제 공제 | 경제적 불만에 대한 반응 |
| Day 5 | 외부 감사관 불시 검문 | 감사관 NPC 판별 난이도 최고조 |

### 6.4. 아이템 사용 효과 (심문 중)

| 아이템 | 사용 효과 |
| :--- | :--- |
| 고농축 에너지 드링크 | 현재 NPC 대화 턴 +2 (추가 질문 기회) |
| 최고급 연초 | 현재 NPC 스트레스 즉시 +50 |
| 불법 최면 어플 | 다음 1회 질문에 대해 AI 진실 답변 강제 (`[TruthItem] On` 프롬프트 주입) |

## 8. 엔딩 분기 시스템

5일차 종료 시 `SettlementController.DetermineEndingType()`이 두 개의 누적 수치로 엔딩을 결정합니다.

### 8.1. 엔딩 판별 기준 (실제 코드 수치)

| 기준 | 임계값 |
| :--- | :--- |
| 비자금 (`TotalBribe`) | **≥ 2,000G** 달성 여부 |
| 신뢰 포인트 (`trustPoint`) | **≥ 15점** 달성 여부 |

### 8.2. 엔딩 4종 매트릭스

| 비자금 (TotalBribe) \ 신뢰 (trustPoint) | >= 15 | < 15 |
| :--- | :---: | :---: |
| **>= 2,000G** | 👑 영앤리치 (`Ending_True`) | 💀 카르텔 (`Ending_Bad`) |
| **< 2,000G** | 👔 우수 사축 (`Ending_Normal`) | 🗑️ 해고 (`Ending_Fire`) |

| 엔딩 | 조건 | 의미 |
| :--- | :--- | :--- |
| 👑 **영앤리치 (True)** | 비자금 달성 AND 신뢰 달성 | 완벽한 줄타기 — 최상위 결말 |
| 💀 **카르텔 (Bad)** | 비자금 달성 AND 신뢰 미달 | 보안 붕괴 — 본사에 점령당함 |
| 👔 **우수 사축 (Normal)** | 비자금 미달 AND 신뢰 달성 | 청렴하지만 가난한 충견 |
| 🗑️ **해고 (Fire)** | 비자금 미달 AND 신뢰 미달, 또는 **자금 마이너스** | 능력 부족 또는 파산 |

> **파산 조건 우선 처리**: 자금이 마이너스가 되면 어떤 조건과 무관하게 즉시 `Ending_Fire` 강제 전환.
> 엔딩 분기는 `CutScene`(연출) → `EndScene`(결과 화면) 순서로 재생됨.

## 9. LLM 프롬프트 엔진

본 프로젝트의 핵심 기술 차별점. 단일 고정 프롬프트가 아닌 **6개 레이어를 런타임에 동적으로 조립**합니다.

### 9.1. 프롬프트 조립 순서 (MakePrompt)

| 레이어 | 역할 |
| :---: | :--- |
| **Layer 1** | 공통 규칙 — 세계관, JSON 출력 강제, 탈옥 방어 기본 규칙 |
| **Layer 2** | NPC 기본 정보 — 이름, 종족, 직책, 목적, 스트레스 (KnowledgeType 분기) |
| **Layer 3** | NPC 타입 지침 — 유형별(진짜/위장/감사관/밀수꾼) 특수 가드레일 |
| **Layer 4** | 종족 페르소나 — 종족별 진짜/가짜 연기 지침 (isPass 분기) |
| **Layer 5** | 일일 특이사항 — 당일 메모 (Real/Fake 분기) |
| **Layer 6** | 감정 상태 — 지능 행동 지침 + 스트레스 구간별 감정 지침 |
| **Optional** | TruthItem — `[TruthItem] On` 태그 (아이템 사용 시 1회 주입) |

### 9.2. 4중 탈옥 방어 레이어

| 레이어 | 구현 방식 |
| :--- | :--- |
| **Meta Command Guard** | 탈옥 키워드 Regex 탐지 → `META_ALERT` 삽입 |
| **TURN_REMINDER** | 매 턴 규칙 재주입 (장기 대화에서 지침 희석 방지) |
| **Player Message Wrapping** | `<player_speech>` 태그로 유저 입력 명확히 구분 |
| **System Re-injection** | 매 턴 System Prompt 최신 상태로 갱신 |

### 9.3. Data-Driven 구조
Unity `ScriptableObject`에 저장된 기획 수치(스트레스 구간, 지능 행동 지침 등)가 프롬프트 변수로 치환됨 → **코드 수정 없이 기획 데이터 변경이 AI 행동에 즉시 반영.**

## 10. 씬 구성 & 기술 스택

### 10.1. 씬 구성 (총 5개)

| 씬 | 역할 |
| :--- | :--- |
| `MainScene` | 타이틀 화면 — 새 게임 / 이어하기 / 설정 |
| `LoadingScene` | 씬 전환 중간 로딩 화면 |
| `GameScene` | 핵심 심문 게임플레이 |
| `SettlementScene` | 일일 정산 + 자판기 상점 |
| `EndScene` | 엔딩 결과 화면 (재시작/메인/종료) |

> CutScene은 독립 씬이 아닌 `CutSceneManager`가 EndScene 진입 전에 재생.

### 10.2. 매니저 아키텍처 요약

| 매니저 | 역할 |
| :--- | :--- |
| `GameManager` | 핵심 게임 루프, GameState FSM(Intro/Playing/Finishing/Outro) |
| `DataManager` | SessionData·SaveData·StaticData 관리, JSON 저장/로드 |
| `UIManager` | 전역 팝업 제어 (Pause, Shop, Permit 등) |
| `AudioManager` | BGM, SFX 재생 및 볼륨 제어 |
| `VFX_Managers` | 화면 흔들기, NPC 이동, 도장, 불꽃 등 VFX 이벤트 버스 |
| `NPCChatManager` | 프롬프트 조립 및 AI 통신 오케스트레이션 |
| `OpenAIClient` | OpenAI API 통신, 응답 파싱, 탈옥 방어 |
