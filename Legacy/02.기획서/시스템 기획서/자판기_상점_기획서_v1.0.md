## 3. 경제 시스템 설계 원칙 (Economic Balance Rules)
본 항목은 FGT(4/17) 목표인 5일차 루프의 유기적 순환을 위한 재화 설계 기준을 정의함.

### 3.1. 재화 획득 및 지출 기준
매일 5명의 NPC가 방문하는 5일 차 루프(총 25명)를 기준으로 한 수치 명세.

| 항목 | 금액 (G) | 발생 조건 및 상세 설명 |
| :--- | :--- | :--- |
| **기본 심문 성공** | **+100** | 정확한 판정(PASS/EJECT)으로 NPC 심문 완료 시 지급 |
| **오판 (실수)** | **0** | 판정 실패 시 수당 미지급 |
| **뇌물 수수** | **+200** | 뇌물 수락 후 PASS 시 획득 (수당 미지급, 뇌물 스택 +1, 보안 평점 0점) |
| **일일 고정 지출** | **-250** | 매일 정산 시 '사내 복지 기금 및 방세' 명목 강제 차감 |
| **감사관 통과 벌금** | **-400** | 위장한 감사관의 함정에 넘어가 PASS 시 다음 날 정산에서 차감 |

### 3.2. 최종 엔딩 분기 조건 (5일 차 종료 시)
누적된 '보안 평점'과 '뇌물 스택'을 대조하여 엔딩을 결정함.

| 엔딩 명칭 | 발동 조건 (Threshold) | 기획 의도 및 결과 요약 |
| :--- | :--- | :--- |
| **👑 영앤리치 (True)** | 평점 18↑ AND 뇌물 4↑ | **"완벽한 줄타기"** - 보안 유지와 비자금 챙기기 동시 달성 |
| **💀 카르텔 (Bad)** | 평점 17↓ AND 뇌물 4↑ | **"보안 붕괴"** - 무분별한 뇌물 수수로 본사 점령당함 |
| **👔 우수 사축 (Normal)** | 평점 18↑ AND 뇌물 3↓ | **"가난한 충견"** - 보안은 지켰으나 돈이 없어 회사에 예속됨 |
| **🗑️ 해고 / 파산 (GameOver)**| 평점 17↓ OR 잔액 0↓ | **"가차 없는 토사구팽"** - 능력 부족 또는 벌금 누적으로 파산 |

## 5. 상점 진입 및 종료 시퀀스 (Shop Flow & Logic)

### 5.1. 진입 트리거 (Trigger)
- **위치**: 메인 씬 종료 후 전환되는 '사내 OS 정산 화면(바탕화면)'.
- **실행**: 바탕화면에 배치된 **자판기.exe** 아이콘 더블 클릭 시 상점 창 호출.

### 5.2. 종료 및 다음 일차 전환
- **상점 닫기**: 상점 모달 창 우측 상단 [X] 버튼 클릭 시 정산 화면 복귀.
- **퇴근(턴 종료)**: 정산 화면의 '일일정산.exe' 내 **[퇴근]** 버튼 클릭 시 다음 일차 심문 씬으로 이동.

## 6. UI/UX 연출 및 예외 처리 (Interface & Edge Case)

### 6.1. UI 디자인 및 상호작용
- **디자인 톤**: Windows 95 스타일 (실버 배경 #c0c0c0, 네이비 타이틀바 #000080).
- **구매 연출**: 클릭 시 재화 차감 SFX 발생 및 해당 슬롯에 **'SOLD OUT'** 오버레이 출력.
- **특수 연출**: '최면 어플' 사용 시 0.5초간 화면 노이즈 및 해킹 콘솔 글리치(Glitch) 이펙트 노출.
- **오디오**: 상점 전용 BGM 출력을 위한 독립적인 오디오 소스 필요.

### 6.2. 예외 처리 (Edge Case)
| 상황 (Context) | 처리 로직 (Logic) |
| :--- | :--- |
| **재화 부족 시** | 클릭 시 구매 불가 SFX 및 "잔액이 부족합니다" 시스템 경고창 팝업. |
| **파산(0G 미만) 시** | 정산 종료 후 즉시 '실업급여는 없습니다(GameOver)' 씬으로 강제 전환. |
| **최면 어플 오입력** | 규격 외 텍스트 입력 시에도 아이템 효과는 사용된 것으로 간주 (환불 불가). |
| **5일차 종료 시** | 최종 정산 후 상점 진입 버튼 비활성화, 즉시 엔딩 분기 로직 실행. |
