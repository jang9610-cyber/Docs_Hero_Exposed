# 🛡️ [용사님, 들켰죠?] 프로젝트 통합 요약본 (Project Omnibus Summary)

본 문서는 프로젝트 **[용사님, 들켰죠?] (Hero Exposed)**의 모든 기획, 기술, 프로젝트 관리 정보를 하나로 통합한 마스터 요약본입니다. 프로젝트의 핵심 가치부터 현재의 기술적 성과, 그리고 미래의 로드맵까지 상세히 기술합니다.

---

## 📑 1. 프로젝트 개요 (Overview)

- **프로젝트 명**: 용사님, 들켰죠? (Hero Exposed)
- **팀명**: 개소라이팅 (GASOLighting)
- **장르**: 블랙 코미디 추리 심문 시뮬레이션
- **플랫폼**: PC (16:9 고정 해상도 기반 정밀 UX)
- **핵심 컨셉**: "AI와의 정교한 심리전을 통한 몬스터 위장 용사 색출"

### 🌍 세계관 및 스토리
전쟁 대신 경제 보복과 M&A로 세계를 정복한 초거대 기업 **'마왕물산'**의 시대. 실업자가 된 용사들은 마왕 회장을 처치하기 위해 몬스터 가죽을 뒤집어쓰고 본사 '출입 대기자'로 위장 잠입합니다. 플레이어는 본사 보안 데스크의 심문관이 되어, 이들의 정체를 밝혀내거나 뇌물을 받고 통과시켜주는 위험한 선택의 기로에 서게 됩니다.

## 🎮 3. 핵심 게임 시스템 (Game Systems)

### **3.1. 핵심 플레이 루프 (Core Loop)**
1.  **데이터 대조**: 지원자의 사원증 정보와 룰북(보안 지침서)의 상식 대조.
2.  **심화 심문 (Interrogation)**: **제한된 질문 횟수(기본 5회)** 내에서 자유로운 대화를 통한 모순 추출.
3.  **최종 판정 (Verdict)**: 승인 도장 또는 하수구 처단 스위치 작동.
4.  **정산 및 상점**: 정산 결과에 따른 비자금 획득 및 상점(어둠의 상점) 아이템 구매.

### **3.2. 심리 압박 지수 (Pressure Index)**
AI NPC는 고유한 **스트레스 임계치**를 가집니다. 유저가 논리적 모순을 지적하거나 '진실의 엘릭서' 같은 아이템을 사용하면 수치가 상승하며, 일정 수준 이상 도달 시 **땀 흘림, 텍스트 진동, 스프라이트 변화(Flustered/Panic)** 등의 실시간 피드백을 제공합니다.

## ⚙️ 4. 기술적 특징 및 돌파구 (Technical Highlights)

### **4.1. 3계층 프롬프트 엔진**
AI의 환각 현상을 방지하고 논리적 무결성을 유지하기 위해 계층형 구조를 채택했습니다.
- **[정체성]**: NPC의 성격, 말투, 비밀 설정 (Identity)
- **[지식]**: 룰북 정보 및 개인 이력 데이터 (Knowledge - JSON)
- **[심리]**: 실시간 스트레스 수치 및 유저와의 관계 (Psychology)

### **4.2. 기술적 혁신: JSON 응답 강제 (ResponseFormat)**
AI가 시스템 수치를 가변적으로 조정할 때 발생하던 파싱 오류를 해결하기 위해 API 레벨에서 형식을 강제합니다.

#### **[핵심 코드 예시 (C#)]**
```csharp
// Gemini API 호출 시 응답 형식을 JSON으로 강제하여 파싱 안정성 확보
var request = new CreateChatCompletionRequest
{
    Model = "gemini-pro",
    Messages = dialogueHistory,
    ResponseFormat = new ResponseFormat 
    { 
        Type = ResponseType.JsonObject // JSON 응답 강제 (기술적 돌파구)
    }
};
```

### **4.3. 메모지 기반 가이드라인 시스템**
유저가 질문을 고착하거나 무한 루프에 빠지는 것을 방지하기 위해, 매일 변경되는 **'일일 특이사항(Notice)'** 데이터를 시스템 프롬프트에 실시간 연동하여 NPC가 논리적 방어 기제를 갖추도록 설계했습니다.
