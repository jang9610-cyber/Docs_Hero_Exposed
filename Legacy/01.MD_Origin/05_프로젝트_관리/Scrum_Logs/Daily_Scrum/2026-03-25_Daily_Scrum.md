## 📢 주요 논의 및 결정 사항

### **1. 유니티 UI 상세 배치 업무 분담 (Dev ↔ Design)**
- **논의 요지**: 
    - 개발자(강다영)가 UI 기획서를 바탕으로 이미지 배치를 시도했으나, 예시 이미지와 실제 배치 간의 미세한 오차가 발생하는 등 디테일 튜닝에 공수가 많이 들어가는 상황임.
- **결정 사항**: 
    - **디테일 배치 업무 이관**: 현 단계의 UI 배치는 기본 연결 위주로 진행하고, 상세한 위치 조정 및 이미지 삽입은 **장수영(디자인) 담당자**가 직접 유니티 프로젝트에서 작업하기로 함.
    - **수치 업데이트**: 장수영 담당자가 직접 배치 후, 최종 확정된 수치를 UI 기획서에 반영하여 문서의 최신성을 유지함. (시점: 연출 기획서 작업 완료 후)

### **2. 캐릭터 원화 제작 일정 및 기획 전환 (Design ↔ PM)**
- **논의 요지**: 
    - AI 활용과 손그림 병행 과정에서 캐릭터 원화 제작에 예상보다 많은 시간이 소요되고 있음. 
    - 일정 지연을 방지하기 위해 기획 파트의 핵심인 '연출 기획서' 작성을 우선순위에 두기로 함.
- **결정 사항**: 
    - **원화 작업 Cut-off**: 캐릭터 원화 작업은 **금일(03-25)까지만 진행**함. 모든 캐릭터가 완성되지 않더라도 내일(03-26)부터는 무조건 **'연출 기획서'** 작성 단계로 전환함.
    - **프로토타이핑 전략**: 당장 필요한 이미지는 현재 제작된 분량과 임시 이미지를 활용하여 배치하고, 캐릭터 원화의 디테일 수정 및 추가 작업은 기획 파트가 대략적으로 마무리된 후 일정 여유에 맞춰 진행함.

### **3. AI 대화 세션 및 컨텍스트 규격 최적화 (Dev ↔ PM)**
- **논의 요지**: 
    - AI 대화 시스템의 호출 효율성을 높이고 불필요한 데이터 구조를 단순화하기 위함.
- **결정 사항**: 
    - **제어 변수 삭제**: 대화 요약(Summarization) 서브루틴을 당장 사용하지 않으므로, 이에 종속된 `Force_System_Prompt` 변수를 규격에서 삭제함.
    - **데이터 주입 통합**: '종족 페르소나'와 '개인 신상 정보'를 굳이 2단계로 나누어 호출/저장할 필요가 없으므로, 이를 하나의 통합된 데이터 패키지로 호출하여 개발 편의성과 응답 속도를 개선함.
