## 🎙️ 미팅 주요 내용
### **1. 개발 및 버그 수정 (강다영)**
- **진행 상황**:
    - **크레딧 버그 수정**: 메인 메뉴에서 크레딧 버튼이 작동하지 않던 버그를 스크립트 수정을 통해 해결 완료.
    - **세이브/로드 시스템**: 이어하기(Continue) 기능을 위한 데이터 저장 및 불러오기 로직 구현 착수.
    - **인게임 디버깅**: 플레이테스트를 통해 발견된 추가 버그(스크립트 기반) 수정 진행 중.
- **주요 과제**:
    - **돈 차감 버그**: 어떤 버튼을 누르든 돈이 10,000원씩 깎이는 치명적인 버그 원인 파악 및 수정 필요.
    - 이어하기 버튼의 UI 연동 및 로직 안정화.

### **2. 기획 및 디자인 (장수영)**
- **진행 상황**:
    - **UI/애니메이션**: 메인 씬 UI 상세 조절 및 크레딧 프리팹 이미지 에셋 제작/애니메이션 적용 완료.
    - **에셋 제작**: 아이템 3종 이미지 에셋 및 배경 레이어 이미지 추가 제작.
    - **일시정지 기능**: ESC 키 조작 시 나타날 일시정지 오버레이 UI 기획 (계속하기, 설정, 메인메뉴 버튼 포함).
- **디자인 최적화**:
    - 기존 이미지 에셋을 재활용하여 일시정지 화면의 시각적 통일성 유지.

### **3. API 보안 및 배포 전략 (공통)**
- **보안 이슈**: 클라이언트 빌드 시 API 키가 노출될 위험(ILSpy, dnSpy 등) 확인.
- **대응 방안**: 
    - 백엔드 서버 연동이 가장 이상적이나, 대안으로 개인 API 키 사용 권장 또는 유저가 직접 자신의 API 키를 입력할 수 있는 필드 제공 검토.
    - 보안 문제가 해결될 때까지 외부 배포 및 유저 QA 일정 조율 필요.
