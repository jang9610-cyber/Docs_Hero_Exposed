# [Role] 개별 직무 로드맵: 개발 (Dev) (v1.2)

본 문서는 강다영 레이서(개발)의 실무 구현 마일스톤 및 완료 기준(DoD)을 정의합니다. **2D URP 인프라** 구축과 **대화형 AI 엔진 정밀도 확보**에 집중하여, 개발자가 자율적으로 다음 단계를 체크하고 이행할 수 있도록 구조화되었습니다.

> **💡 개발자 안내**: 본 로드맵의 각 태스크 완료 시 `[DONE]` 표시를 하고, 상단 완료 기준(DoD) 충족 여부를 자가 검증한 뒤 PM에게 확인을 요청합니다.

---

## 🏗️ 1단계: 시스템 기반 및 데이터 인프라 (Base Infrastructure)
*목표: 프로젝트의 유지보수성과 확장성을 담보하는 물리적 코드 구조 완성*

1.  **[DONE] Unity & Environment 설정**
    - Unity 2022.3 LTS + URP 2D Renderer 전환 및 Git 브랜치 전략(Main/Dev/Feature) 수립.
2.  **몬스터 및 NPC 데이터 아키텍처 (Master Data)**
    - `ScriptableObject`를 활용한 NPC 마스터 데이터 시스템 구현 (종족명, 페르소나, 수치 데이터 등).
    - **완료 조건**: 신규 NPC 추가 시 코드 수정 없이 SO 에셋 생성만으로 데이터 반영 가능 여부.
3.  **세이브 및 상태 관리 시스템 (Persistence)**
    - `DataManager`를 통한 일차별 비자금(`Slush_Fund`), 고과점수(`TrustPoint`), 진행도(`SaveData`) 저장 엔진 구축.
    - **완료 조건**: 게임 재시작 시 이전 날짜의 획득 재화와 고과 상태가 정확히 로드되어야 함.
4.  **씬 오케스트레이터 (Scene Flow)**
    - `SceneManager` 기반 비동기 전환(Title ↔ Main ↔ NextDay ↔ Result) 및 데이터 전달 환경 구축.

---

## 🧠 2단계: 핵심 기능 연동 및 AI 정밀도 고도화 (Core Logic)
*목표: 심문 엔진의 핵심 게임성과 AI 대화의 몰입감(High-Fidelity) 확보*

1.  **AI 대화 엔진 및 보안 프롬프트 (Persona Guard)**
    - Gemini/GPT API 연동 및 스트리밍 수신 모듈 완성.
    - 사고 과정(CoT) 마스킹 처리 및 유도 심문 방어용 전용 시스템 프롬프트 레이어링 적용.
    - **완료 조건**: AI가 사고 과정을 노출하지 않고, 유저의 유도 질문에도 페르소나를 유지해야 함.
2.  **NPC 런타임 인터랙티브 상태 기계**
    - 실시간 의심도/스트레스 수치 연산 및 임계점(`Threshold`) 판정 로직.
    - 상태 변화에 따른 **표정 스프라이트 즉시 교환** 트리거 시스템 (Normal/Panic/Slip).
3.  **UI 엔진 결합 및 데이터 바인딩**
    - 디자인팀의 최종 UI 소스(사원증, 대화창 등) 프리팹 구성 및 런타임 데이터 매핑 시스템.
    - **완료 조건**: NPC 데이터 SO의 내용이 실제 게임 내 사원증 UI에 정확히 바인딩되어 출력됨.

---

## 💰 3단계: 경제 시스템 구현 (Economy)
- **상점 로직**: 아이템 구매 로직 및 재화 소모 시스템 구축.
- **데이터 소비**: 구매한 아이템(심문권 등)의 효능이 다음 날 로직에 실제 반영되도록 연동.
- **상세 정산**: 하루 성과(뇌물, 벌금)를 합산하여 비자금에 누적 시키는 정산 엔진 완성.

---

## ✨ 4단계: 연출 및 최적화 (Polishing)
- **URP FX**: 2D Lighting 및 `Vignette/Bloom` 포스트 프로세싱 효과 적용.
- **Canvas 및 쉐이더 연동**: 디자인팀의 최종 소스를 UI Canvas 레이어로 구성하고 URP UI 쉐이더/애니메이션 연동.
- **사운드 통합**: 판정 SFX 및 배경음악 믹싱 엔진 연동.

---

## 🧪 5단계: 컨텐츠 확장 및 엔딩 (Mastering)
- **데이터 통합**: 1-5일차 전수 시나리오 데이터 임포트 및 예외 처리 QA.
- **멀티 엔딩**: 비자금/보안평점 2x2 매트릭스에 따른 최종 엔딩 씬 자동 전환 로직 완성.
- **최적화**: SRP Batcher 점검 및 최종 빌드 안정성 테스트.
