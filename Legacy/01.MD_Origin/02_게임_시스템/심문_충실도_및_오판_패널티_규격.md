# ⚖️ 심문 충실도 및 오판 패널티 규격 (Interrogation Confidence) (v1.0)

본 문서는 플레이어가 AI와 충분한 대화를 나누지 않고 임의로 판정(찍기 플레이)하는 것을 방지하기 위한 **심문 충실도(Confidence) 시스템**과 이에 따른 **패널티 가중치**를 정의합니다.

---

## 🏗️ 1. 심문 충실도 정의 (Confidence Tiers)

*플레이어가 현재 NPC에게 던진 **누적 질문 횟수(`TurnCount`)**를 기준으로 판정의 신뢰도를 결정합니다.*

| 충실도 등급 | 질문 횟수 (`TurnCount`) | 오판 패널티 가중치 | UI 표시 (Label) |
| :--- | :--- | :--- | :--- |
| **🚨 낮음 (Risky)** | 0 ~ 1회 | **3.0x (3배 감점)** | **근거 부족 (High Risk)** |
| **⚠️ 보통 (Uncertain)** | 2회 | **2.0x (2배 감점)** | **조사 중 (Caution)** |
| **✅ 높음 (Reliable)** | 3회 이상 | **1.0x (표준 감점)** | **판정 가능 (Ready)** |

---

## 🎲 2. 통합 인사 고과 산출 로직 (Integrated Scoring)

*단순한 명수 구분을 넘어, 모든 판정 결과는 실시간으로 `TrustPoint (0~100)`에 반영됩니다.*

### 2.1. 판정별 고과 변동표
| 판정 상황 | 재화 변동 (Gold) | 인사 고과 변동 | 상세 내용 |
| :--- | :--- | :--- | :--- |
| **정판 성공 (정답)** | **+100 G** | **+2점** | 용사 검거 또는 정상 몬스터 통과 |
| **판정 실패 (오판)** | **0 G** | **-10점** | 오판 시 보상 미지급 및 고과 감점 |
| **감사관 뇌물 수수** | **-1,000 G** | **-50점** | 보안 감사 위반 (치명적 패널티) |

### 2.2. 인사 고과(`TrustPoint`) 관리 규칙
- **완주 보장 (No Game Over)**: 인사 고과가 0점 이하로 떨어지더라도 게임은 중단되지 않으며, 무조건 5일차(`Day 5`) 정산까지 진행됩니다.
- **감점 누적**: 0점 미만으로도 점수가 내려갈 수 있으며(마이너스 점수 가능), 이는 최종 엔딩에서 **'파면'** 또는 **'처형'** 등의 최악의 결과로 직결됩니다.
- **최종 환산**: 엔딩 시점의 `TrustPoint` 총합에 따라 최종 S~F 등급과 엔딩 시네마틱이 결정됩니다.

## 🖼️ 3. UI/UX 연출 가이드 (User Feedback)

### 3.1. 판정 신입 전 경고 (Confirm Gate)
- **발동 조건**: 충실도가 **'낮음'** 혹은 **'보통'**일 때 판정 버튼(수락/거절) 클릭.
- **메시지 내용**: 
    - "아직 조사가 충분하지 않습니다! 지금 판정하면 **최대 3배의 감점**을 받을 수 있습니다. 강행하시겠습니까?"
    - `[ 계속 심문하기 ]` / `[ 강제 판정하기 ]` 선택지 제공.

### 3.2. 신뢰도 게이지 UI
- 판정 버튼 상단에 3단계로 변화하는 LED 또는 게이지 바를 배치하여, 질문을 던질 때마다 색상이 변하는 시각적 피드백 제공 (빨강 → 노랑 → 초록).

## 📐 4. 기술 연동 가이드 (Developer Interface)

### 4.1. 주요 변수 및 데이터 타입
| 변수명 | 타입 | 초기값 | 설명 |
| :--- | :--- | :--- | :--- |
| `TurnCount` | `int` | 0 | 현재 NPC에게 수행한 질문 총합 |
| `PenaltyMultiplier` | `float` | 3.0f | `TurnCount`에 따라 실시간으로 변하는 가중치 |
| `IsEvidenceFound` | `bool` | `false` | 특수 태그 발견 여부 플래그 |
