# 🏆 엔딩 마스터 규격 (Ending Master Specification) (v1.1)

본 문서는 플레이어의 5일간의 심문 성과에 따른 엔딩 판정 로직, 산출 데이터 공식, 그리고 최종 결과 리포트(UI) 규격을 통합 정의합니다.

---

## 🏁 1. 엔딩 트리거 및 판정 (Ending Trigger)

### 1.1. 판정 시점 (Trigger)
- **5일차 완주 보장**: 모든 플레이는 **5일차(`Day 5`)의 마지막 NPC 판정**이 완료된 직후 엔딩 시퀀스로 진입합니다. (중도 탈락 없음)

### 1.2. 엔딩 2x2 매트릭스 (Final Decision)
*최종 누적 **보안 평점(Security Score)**과 **뇌물 스택(Bribe Stack)**의 조합으로 결정됩니다.*

| 구분 | 💰 뇌물 부족 (`Stack <= 3`) | 💰 뇌물 풍족 (`Stack >= 4`) |
| :--- | :--- | :--- |
| **📉 보안 저조**<br>(`Score <= 17`) | **[END_01] 해고 / 파산 (Game Over)**<br>능력 부족 또는 벌금 누적으로 인한 퇴사 | **[END_03] 💀 카르텔 (Bad)**<br>뇌물에 눈이 멀어 본사가 점령당함 |
| **📈 보안 우수**<br>(`Score >= 18`) | **[END_02] 👔 이달의 우수 사축 (Normal)**<br>보안은 지켰으나 여전히 가난한 노예 | **[END_04] 👑 영앤리치 (True Ending)**<br>보안과 실익을 모두 챙긴 완벽한 엑시트 |

---

## 📐 2. 수치 데이터 산출 로직 (Calculation Logic)

### 2.1. 최종 보안 등급 (`FinalSecurityRank`) 공식
- **공식**: `(TrustPoint * 0.7) + (TotalHeroCaught * 10) - (TotalWrongDecision * 5)`
- **등급**: **70점 이상** 시 'High(우수)', **70점 미만** 시 'Low(저조)'로 판정.

### 2.2. 사용 변수 정의 (Variables)
| 변수명 | 설명 | 비고 |
| :--- | :--- | :--- |
| `TrustPoint` | 최종 시점의 인사고과 (0~100) | 누적 감점/가산 반영 |
| `SecurityScore`| 최종 보안 평점 (최대 25점) | 엔딩 판정의 핵심 축 (18점 기준) |
| `BribeStack` | 5일간 누적된 뇌물 스택 (최대 25) | 엔딩 판정의 핵심 축 (4스택 기준) |
| `TotalBribe` | 5일간 모은 총 누적 비자금 | UI 표시용 |
| `TotalHeroCaught` | 검거에 성공한 총 용사 수 | 핵심 가산점 요소 |
| `TotalWrongDecision`| 오판 및 보안 사고 총 횟수 | 핵심 감점 요소 |

---

## 🖼️ 3. 결과 리포트 UI 데이터 (Result Report)

*최종 엔딩 시네마틱 전, 성적표 화면에 출력될 항목입니다.*

1.  **"성실한 노예의 고과"**: 최종 `TrustPoint` 표시
2.  **"총합 골드"**: `TotalBribe` 표시
3.  **"검거한 용사수"**: `TotalHeroCaught` 명
4.  **" 업무 평가 점수"**: `TotalWrongDecision` 점
5.  **"최종 평가 등급"**: 계산된 `FinalSecurityRank` (A/B/C/F)
