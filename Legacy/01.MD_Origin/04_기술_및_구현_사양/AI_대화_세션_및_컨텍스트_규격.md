# 📝 대화 문맥(Context) 데이터 시트 정의 (Context Data Spec)

본 문서는 개발자(강다영)의 대화형 AI 컨텍스트 구현을 돕기 위해, 마스터 데이터 시트(XLSX)에 반영될 **심문 대화 기억 및 관리 로직**의 데이터 사양을 정의합니다.

---

## 📊 1. 컨텍스트 제어 데이터 (Context Control Table)

| **Variable-string** | **Type** | **Default Value** | **Description** |
| :--- | :--- | :--- | :--- |
| `Context_Max_Turns` | `int` | `10` | 질문 제한(5회)에 대비한 [유저 5 + AI 5] 누적 전송 수치 |
| **`Context_Reset_Trigger`** | `string` | **`On_Decision_Made`** | **핵심**: NPC 판정(통과/방출) 클릭 시 히스토리 완전 삭제 |
| `History_Method` | `string` | **`Full_Session`** | **추천**: 요약 없이 세션 전체 데이터 전송 (정확성/속도 우선) |

---

## 🧠 2. 심문 로직 및 단계별 구현 (Interrogation Logic & Roadmap)

대화 컨텍스트는 토큰 효율보다 **AI의 정확한 논리 판단**과 **응답 속도**를 최우선으로 하여 단계별로 구현합니다.

### **2.1. 1단계: 전체 세션 데이터 전송 (현재 적용 방식)**
*   **방식**: 대화 요약(Summarization) 서브루틴을 거치지 않고, 모든 히스토리(`List<ChatMessage>`)를 그대로 API에 전송.
*   **채택 사유**:
    1.  **정확성**: 5회 제한의 짧은 대화에서는 요약보다 원본 텍스트가 AI의 모순 포착 및 할루시네이션 방지에 유리함.
    2.  **응답 속도 (Latency)**: 요약 API를 별도 호출(2번 연산)할 경우 인게임 대기 시간이 길어지는 현상 방지.
    3.  **토큰 비용**: 5턴 내외의 짧은 세션은 전체 전송 시에도 토큰 부담이 크지 않음.

### **2.2. 세션 라이프사이클 (Session Lifecycle)**
1.  **세션 시작 (New NPC)**:
    *   신규 NPC 등장 시 인게임 `List<ChatMessage>`를 `Clear()`함.
    *   **통합 데이터 주입 (Data Injection)**: 종족 페르소나와 개별 신상 정보(이름, 직업, 목적 등)를 하나의 패키지로 통합하여 첫 대화 시작 시 주입.
2.  **질문 진행 (Under Questioning)**:
    *   질문 제한(5회) 이내의 모든 발화를 리스트에 누적하여 매 호출 시 전체 보냄.
3.  **세션 종료 및 초기화 (Reset)**:
    *   **판정 완료 시점**: [통과] 또는 [방출] 버튼 작동 시 히스토리를 즉시 초기화하여 세션 간 간섭 차단.

## 🛠️ 3. 개발자 가이드 (Dev Implementation)

1.  **데이터 저장**: 유니티 내부에 `List<ChatMessage>` 클래스 형태로 대화 데이터를 파싱하여 유지합니다.
2.  **데이터 전송 구조 (예시)**:
    ```json
    {
      "messages": [
        {"role": "system", "content": "너는 비밀을 숨긴 오크 정비공이다. 질문에 대해 방어적으로 답변하라..."},
        {"role": "system", "content": "NPC Profile Data: {Name: 그로그락, Age: 35, Job: 정비공, Purpose: 무기 수리...}"},
        {"role": "user", "content": "당신 이름이 뭐지?"},
        {"role": "assistant", "content": "나는 그룩... 아니, 그로그락이다."},
        {"role": "user", "content": "거짓말 마, 아까는 30살이라고 했잖아!"}
      ]
    }
    ```
    *※ 위와 같이 히스토리 전체를 하나의 요청으로 서버에 전송해야 AI가 모순을 인지할 수 있습니다.*
