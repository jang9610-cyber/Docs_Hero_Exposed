# 💾 세이브 및 상태 관리 규격 (Save & State Management Spec) (v1.2)

본 문서는 프로젝트 `[용사님, 들켰죠?]`의 게임 데이터 영속성(Persistence)과 실시간 게임 상태(Runtime State) 관리를 위한 기술 규격서입니다. 개발자(강다영)는 본 규격에 따라 세이브 시스템과 런타임 데이터 매니저를 구현합니다.

---

## 📊 1. 데이터 분류 및 관리 주체

데이터는 그 성격에 따라 **영구 저장 데이터(Global)**와 **실시간 세션 데이터(Session)**로 구분하여 관리합니다.

### 1.1. 영구 저장 데이터 (Global Save Data)
*앱을 종료하고 다시 시작해도 유지되어야 하는 핵심 데이터입니다.*

| 변수명 | 타입 | 저장 시기 | 상세 설명 |
| :--- | :--- | :--- | :--- |
| **`DayState`** | `int` | 일차 종료 시 | 현재 진행 중인 게임 일차 (1~5일) |
| **`TrustPoint`** | `int` | 판정 직후 | **[필수]** 실시간 인사 고점 (초기값: 100) |
| **`TotalBribe`** | `int` | 판정 직후 | 현재까지 누적된 총 비자금 (뇌물 합계) |
| **`TotalHeroCaught`** | `int` | 판정 종료 시 | 위장 잠입한 용사를 적발해낸 누적 명수 |
| **`TotalWrongDecision`**| `int` | 판정 종료 시 | 오판(용사 통과 또는 몬스터 방출) 누적 횟수 |
| `GameClearCount` | `int` | 엔딩 도달 시 | 전체 게임 클리이 횟수 |

### 1.2. 실시간 세션 데이터 (Ephemeral/Runtime Data)
*현재 대화 중인 NPC와의 세션 동안만 유효하며, 판정 후 초기화됩니다.*

| 변수명 | 타입 | 초기값 | 상세 설명 |
| :--- | :--- | :--- | :--- |
| `CurrentStress` | `int` | 0 | 현재 NPC의 실시간 스트레스 수치 (0~100) |
| `SuspicionLevel` | `int` | 0 | 현재 NPC의 거짓말 의심도 (0~100) |
| **`TurnCount`** | `int` | 0 | 현재 NPC에게 던진 질문 횟수 (Max 5) |
| `DialogueLog` | `List<Chat>` | `Empty` | 현재 세션의 대화 히스토리 (AI 전송용) |
| `IsElixirUsed` | `bool` | `false` | 진실의 엘릭서 사용 여부 (세션당 1회) |

## 💾 2. 저장 및 로딩 메커니즘 (Save/Load Logic)

### 2.1. 저장 방식 (Method)
- **포맷**: `JSON` 기반 파일 저장.
- **경로**: `Application.persistentDataPath` (Unity 표준 경로).
- **보안**: 저장 시 단순 텍스트가 아닌 `Base64` 인코딩 또는 체크섬(`Checksum`) 검증 로직 권장.

### 2.2. 저장 시점 (Save Triggers)
1. **판정 완료 (Decision Made)**: [통과] 또는 [방출] 버튼 클릭 즉시 `TotalBribe`와 `TrustPoint`를 저장하여 비정상 종료 시에도 재화 손실 방지.
2. **일차 종료 (Day End)**: 하루 정산 화면이 끝나는 시점에 `DayState`를 가산하여 저장.
3. **옵션 변경**: 설정 변경 시 즉시 저장.
