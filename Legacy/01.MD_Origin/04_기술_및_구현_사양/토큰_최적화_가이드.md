# [가이드] Gemini API 토큰 최적화 및 비용 효율화 전략 (v1.1)

> [!IMPORTANT]
> **본 문서는 2026-03-25 대화 문맥(Context) 전략 결정 사항을 반영하여 업데이트되었습니다. 실제 구현 시 [AI_대화_세션_및_컨텍스트_규격.md]와 교차 확인이 필요합니다.**

---

## 1. 개요
본 문서는 Google Gemini API 활용 시 비용 절감 및 응답 속도(Latency) 향상을 위한 가이드를 제공합니다. 특히 **5턴 제한 심문 시스템**에 최적화된 토큰 관리 전략을 다룹니다.

---

## 2. 입력 데이터(NPC Profile) 구조 최적화

### **A. JSON 키값 단축 (Key Minimization)**
LLM은 JSON의 키(Key) 명칭도 토큰으로 인식합니다. 의미가 변하지 않는 선에서 최대한 단축합니다.
- **AS-IS**: `"SpeciesCharacteristics": "...", "EntryPurpose": "..."`
- **TO-BE**: `"trait": "...", "aim": "..."` (토크나이저 기준 약 30~50% 절감 가능)

### **B. 상시 데이터 시트 주입 (Data Injection)**
- **방식**: 매 호출마다 NPC의 핵심 팩트(이름, 직업 등)를 포함하여 전송하되, 불필요한 장식 문구(Boilerplate)를 배제한 텍스트/JSON 형식을 사용합니다.

---

## 3. 시스템 프롬프트(System Instruction) 경량화

### **A. 역할 정의의 명확성**
중언부언하는 긴 설명은 배제하고, 핵심 페르소나와 제약 조건(Constraint)만 명시합니다.
- "당신은 마왕물산의 보안 검문을 받고 있는 몬스터입니다. 당신은 질문에 대해 반드시 사실만을 말해야 하며..." $\rightarrow$ **"너는 마왕물산 보안 검문을 받는 [몬스터]다. 사원증 데이터에 맞춰 연기하되, 의심도가 높을 때만 당황하라."**

### **B. No-Boilerplate 기법**
"이해했으면 '네'라고 답하세요" 같은 확인성 문구 등을 삭제하고, 출력 형식만 명확히 지정합니다.

- **압축적 페르소나**: 중언부언하는 긴 설명 대신 핵심 성격과 제약 조건만 명시합니다. 5턴의 짧은 세션에서는 긴 지시문보다 명확한 규칙(Constraint) 전달이 토큰 효율에 더 유리합니다.
---

## 4. 대화 이력(Context Window) 관리

### **A. 전체 세션 유지 (Full Session History)**
- **결정 사항**: 5턴 내외의 짧은 심문 세션에서는 **대화 요약(Summarization)을 수행하지 않습니다.**
- **이유**: 요약을 위한 추가 API 호출 비용 및 레이턴시를 방지하고, 원본 텍스트 유지를 통해 AI의 논리 판단 정확도를 높이기 위함입니다.

### **B. 확실한 세션 종료 및 리셋 (Garbage Collection)**
- **핵심 전략**: `Context_Reset_Trigger`를 '판정 완료 시점'으로 설정하여, NPC가 바뀔 때마다 기존 히스토리를 완전히 삭제합니다. 이는 누적 토큰 사용량을 0으로 초기화하는 가장 확실한 최적화 방법입니다.

---

## 5. 생성 파라미터 제어

- **Max Output Tokens**: 심문 답변은 대체로 짧을 것이므로 150~200 토큰 정도로 제한하여 불필요한 긴 답변 생성을 방지합니다.
- **Stop Sequences**: 답변이 종료되어야 하는 지점(예: `\n`, `[End]`)을 명확히 설정하여 토큰 낭비를 막습니다.
