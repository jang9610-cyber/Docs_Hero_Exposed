# 🤝 1인 개발자 지원 및 프로젝트 협업 가이드 (v1.0)
*기획(이남기) 및 디자인(장수영) 파트의 유니티 에디터 내 업무 분담 가이드*

---

## 📌 1. 취지 (Context)
본 프로젝트는 **1인 개발 체제(강다영)**로 운영됨에 따라, 개발자의 로직 구현(Scripting) 부하를 최소화하기 위해 **기획(이남기)** 및 **디자인(장수영)** 파트에서 유니티 에디터 내 리소스 배치 및 데이터 주입 업무를 적극 분담함.

---

## 🛠️ 2. 파트별 유니티(Unity) 업무 분담

### 👔 이남기 (Planner / PM) - 데이터 및 QA 마스터
개발자가 하드코딩하지 않도록, 게임의 '수치'와 '텍스트' 데이터를 유니티 내에서 직접 관리함.
1.  **데이터 에셋 관리**: 엑셀 마스터 시트로부터 추출된 수치 데이터(NPC 스태츠, 확률 등)를 유니티 내 ScriptableObject 또는 CSV에 직접 입력 및 수정.
2.  **Gemini 프롬프트 튜닝**: 유니티 테스트 환경에서 시스템 프롬프트를 직접 수정하며 AI 응답 로직의 무결성을 검증.
3.  **기술적 QA 및 리포팅**: 버그에 대해 유니티 화면을 캡처 및 문서화해서 개발자에게 전달함으로써 디버깅 시간을 단축함.

### 🎨 장수영 (Designer / Planning Support) - 인터페이스 및 연출 마스터
디자인 리소스가 실제 게임 화면에 '정확하게' 배치되도록 유니티 UI 시스템을 직접 세팅함.
1.  **UI Canvas 레이아웃 튜닝**: 기획서 내 수치(픽셀)를 바탕으로 유니티 내 프리팹(Prefab) 내 버튼, 텍스트, 이미지의 위치를 직접 조정.
2.  **애니메이터(Animator) 세팅**: 캐릭터의 3단 표정(Normal → Flustered → Panic) 변화를 위한 애니메이터 상태 전이(Transition) 구성을 직접 담당.
3.  **리소스 삽입 및 관리**: 제작된 스프라이트(Sprite)와 효과음(SFX) 파일을 유니티 프로젝트 폴더에 임포트하고 해당 오브젝트에 직접 매핑함.

---

## 🚀 3. 협업 원칙 (Principles)
1.  **로직은 개발, 데이터는 기획, 외형은 디자인**: 개발자(강다영)는 오직 C# 스크립트 기반의 핵심 로직 구현에만 집중할 수 있는 환경을 조성함.
2.  **에디터 내 직접 수정 권장**: 기획/디자인 파트는 기획서로만 요구사항을 전달하는 대신, 유니티 인스펙터(Inspector) 창에서 직접 값을 변경하며 피드백을 수렴함.
3.  **코드 충돌 방지**: 작업을 시작하기 전 협업 도구(Git 등)를 통해 작업 범위를 소통하여 작업 충돌을 사전 방지함.
