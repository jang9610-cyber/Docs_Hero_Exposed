# [용사님, 들켰죠?] idcarddata 프롬프트 데이터 시트 (v1.0)

## 1. 목적
본 문서는 LLM(Gemini)이 심문 대상의 사원증 정보를 인지하고, 해당 정보의 일관성을 바탕으로 '용사 여부'를 판정받기 위한 **프롬프트 주입용 데이터 구조**를 정의합니다.

## 2. 데이터 필드 정의 (Schema)

| 필드명 | 데이터 타입 | 상세 설명 | 예시 |
| :--- | :--- | :--- | :--- |
| **Name** | String | 심문 대상의 이름 (사원증 표기) | 고르바초프 (본명: 용사 김철수) |
| **Species** | Enum | 종족 (오크, 슬라임, 스켈레톤, 인간 등) | 오크 (Orc) |
| **Position** | String | 마왕군 내 직책 또는 소속 | 제3보급창고 경비병 |
| **Entry_Purpose** | Text | **[중요]** 심층 심문 대상 항목. 출입 목적 상세. | "가족 면회 및 마력석 반납을 위해" |
| **Special_Note** | Text | **[중요]** 심층 심문 대상 항목. 신체적 특징이나 특이 사항. | "왼쪽 어금니가 부러짐, 특이한 문신" |
| **Issue_Date** | Date | 사원증 발급일 | 2026. 02. 15 |
| **Security_Level** | Int | 1~5 단계 보안 레벨 | 1 (일반 병사) |

## 3. 프롬프트 주입 전략 (Prompt Injection)

### **A. 페르소나 설정 예시**
> "너는 지금부터 '오크' 종족의 '제3보급창고 경비병' 연기를 해야 해. 네 사원증에 적힌 출입 목적은 '가족 면회'야. 플레이어가 네 출입 목적에 대해 집요하게 물으면, 사원증에 적힌 내용과 일치하도록 대답하되, 네가 만약 '용사'라면 거짓말을 하는 과정에서 미묘한 심리적 동요(감정/의심도 상승)를 보여야 해."

### **B. 논리 일관성 검증 (Flow)**
1. **사원증 제시**: 플레이어가 사원증을 확인.
2. **질문 단계**: "출입 목적이 무엇입니까?"
3. **LLM 응답**: "가족 면회하러 왔습니다." (정상 응답)
4. **교차 검증**: "가족은 어디에 삽니까? 사원증 발급일이 왜 최근입니까?" 
5. **용사 판정**: 설정된 `Special_Note`와 응답이 충돌하거나, 당황할 시 의심도 상승.
