**[컨셉과 개발의 연결]**

**[향후 추가 계획]**

## 4. 🔄 피드백 반영 이후 개선 사항 & 최종 목표 달성 현황

**[피드백 반영 및 개선점]**
- 최근 발생했던 'NPC 미출력 버그'를 타파하기 위해, 스크립트 생명주기와 실행 순서(Execution Order)를 최적화하여 렌더링 무사고를 달성했습니다. 
- **효율적 협업 프로세스 구축**: 
    - **로직 선구현 (강다영)**: 개발자가 프로그래밍 로직과 임시 배치를 통해 탄탄한 '뼈대'를 먼저 세움.
    - **디자인 후처리 (장수영)**: 그 바통을 이어받아 정식 에셋을 교체하고 픽셀 단위로 UI를 다듬으며 '근육'을 붙이는 후처리 진행.
    - **결과**: 이 병렬 프로세스 덕분에 작업 공간 충돌 없이 씬 연동과 에셋 교체를 진행할 수 있었고, 스타트부터 엔딩까지의 프레임워크를 100% 완성하는 성과를 거둠.

**[최종 목표 대비 현재 단계 현황 (진척률 : 약 75%)]**
현재 프로젝트의 마스터 로드맵 기준 진행 상황입니다.
* **1단계 (프레임워크 및 씬 시퀀닝) - `[완료 / 100%]`**: 타이틀, 인게임, 휴식, 결산으로 이어지는 흐름 및 스켈레톤 구축 완료.
* **2단계 (핵심 로직 연동 및 API 스트리밍) - `[완료 / 100%]`**: Helpy Pro API 연동 및 캐릭터 대화/스트리밍 판정 기반 처리 완료.
* **3단계 (경제 시스템 및 상점 구체화) - `[완료 / 100%]`**: 플레이 루프 내의 상점 로직 및 데이터 재화 순환 연동 완료.
* **4단계 (비주얼 폴리싱 및 대기 연출) - `[진행 중 / 70%]`**: URP 2D 그래픽 개선, 애니메이션 고도화, 일시정지 등 UI 기능 폴리싱 및 데이터 세이브 기능 마무리 단계.
* **5단계 (전체 컨텐츠 통합 및 밸런싱) - `[대기 중 / 0%]`**: 버그 픽스와 4단계 종료 직후 5일 차 스크립트 콘텐츠를 접목하는 마무리 단계.

## 🧩 발표 시연 가이드 (강다영님 담당 파트 📝)

**[시연 목적]**
프로토타입 이후 업그레이드 된 세이브/로드 인프라, 원활히 연동되는 Helpy Pro API 대화 판정, 완성도 높은 UI/UX 시각화 등을 유연하게 발표

**[시연 순서 및 스크립트 가이드]**
1. **타이틀 및 환경설정 (약 1분)**
   - *🎙️ "메인 타이틀 씬의 완성도와 추가된 환경설정 요소들을 보여드리겠습니다."*
   - 메인 화면에 적용된 크레딧 버튼 애니메이션을 재생하여 보여줍니다.
   - 톱니바퀴 아이콘을 클릭하여 '환경설정(Settings)' 창에서 실제 볼륨 조절 기능 활성화 시연.
2. **세이브/로드 데이터 기능 진입 (약 1분)**
   - *🎙️ "현재 새로이 도입한 '이어하기' 기능을 이용해 세이브된 플레이 정보를 불러오겠습니다."*
   - '이어하기' 버튼을 클릭하여 인게임 진입 (혹시 시연용으로 버그가 있다면 빠르게 새 게임으로 진입 우회하여 상황 모면).
3. **핵심 코어 루프(NPC 상호작용) 결합 (약 2분)**
   - *🎙️ "Helpy Pro API 기반 저희의 대화 시스템입니다."*
   - 지정된 NPC(뱀파이어 혹은 구울 등)와 대화를 작동시킨 뒤 자연스러운 실시간 스트리밍 대답을 가볍게 시연하며 보여줌.
4. **투명한 개발 이슈 공유 및 ESC 오버레이 시연으로 마무리 (약 1~2분)**
   - NPC와 상호작용을 하거나 버튼 조작을 통해 일어나는 '10,000원 코인 차감 버그'를 정직하게 보여줍니다. 
   - *🎙️ "현재 이처럼 버튼 이벤트 연결 상의 치명적 충돌이 있어, 집중 디버깅 중에 있다는 점을 솔직하게 공유드립니다."*
   - 그 후 ESC 키를 눌러 현재 설계 중인 '일시정지 오버레이 (계속하기, 환경설정, 메인메뉴로 나가기)' 메뉴 구성을 시각적으로 띄우며 발표를 안정적으로 마무리합니다.
