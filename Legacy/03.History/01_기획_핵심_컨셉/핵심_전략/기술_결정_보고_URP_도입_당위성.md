# [기술 결정 보고] 프로젝트 파이프라인 URP(Universal Render Pipeline) 도입 당위성

본 문서는 `[용사님, 들켰죠?]` 프로젝트의 시각적 품질 극대화와 안정적인 성능 최적화를 위해, 기존 Built-in 파이프라인 대신 **URP(Universal Render Pipeline)**를 도입하기로 결정한 기술적 근거를 명시합니다.

---

## 🏗️ 1. 도입 배경 및 목적
- **비주얼 목표**: lilToon 쉐이더를 활용한 고품질 애니메이션 스타일(Toon Shading) 구현.
- **최적화 목표**: PC 플랫폼에서의 안정적인 프레임 유지 및 로딩 효율화.
- **개발 효율**: Shader Graph 등을 활용한 비주얼 이펙트(VFX) 개발 속도 향상.

---

## 📊 2. URP 도입의 3대 핵심 이유

### **① 압도적인 성능 최적화 (Optimization)**
- **SRP Batcher**: URP의 핵심 기술인 SRP 배처를 통해 동일한 머티리얼을 사용하는 수많은 오브젝트를 한 번에 렌더링(Batching)하여 CPU/GPU 부하를 혁신적으로 줄입니다.
- **2D Renderer**: 2D 게임 전용 렌더러를 제공하여, 기존 3D 방식보다 훨씬 가벼우면서도 정교한 스프라이트 제어가 가능합니다.

### **② 차세대 비주얼 효과 (Visual Excellence)**
- **Integrated Post-processing**: Built-in 대비 훨씬 가볍고 강력한 포스트 프로세싱(Bloom, Color Grading, Chromatic Aberration 등)을 제공합니다. 이는 심문 상황의 긴장감 연출에 필수적입니다.
- **2D Dynamic Lighting**: 실사 라이트가 아닌 스프라이트 전용 조명을 사용하여, 실시간으로 변화하는 애니메이션틱한 그림자를 효율적으로 구현할 수 있습니다.

### **③ 확장성 및 미래 지향성 (Scalability)**
- **Shader Graph 지원**: 코딩 없이 노드 방식으로 쉐이더를 제작할 수 있어, 기획/디자인 파트와의 비주얼 협업이 용이합니다.
- **에셋 호환성**: `lilToon`을 포함한 최신 고퀄리티 에셋들은 URP 환경에서 가장 강력한 기능(최적화된 외곽선 등)을 발휘합니다.

### **④ 왜 2D가 아닌 3D 기반 환경(3D Space)인가?**
- **심층적 공간감 (Z-Axis Utilization)**: 비록 게임의 뷰는 2D이지만, 3D 공간(Z축)을 활용함으로써 캐릭터와 배경 사이의 **패럴랙스(Parallax)** 효과와 **원근감**을 극대화하여 훨씬 생동감 있는 화면을 구성할 수 있습니다.
- **3D 조명과의 시너지**: 2D 오브젝트를 Quad(평면)나 단순 Mesh에 입혀 3D 조명을 받을 경우, 실시간 조명의 **그림자 연산**과 **빛 번짐**을 그대로 활용하여 심화된 분위기(Psychological Tension)를 연출할 수 있습니다.
- **lilToon의 Mesh 기반 사양**: 우리가 사용하고자 하는 `lilToon` 쉐서더는 기본적으로 **MeshRenderer**에서 최적의 성능을 냅니다. 3D 기반 프로젝트로 구축할 때 쉐이더가 제공하는 툰 외곽선과 그림자 처리 기능을 100% 활용할 수 있습니다.
- **역동적인 카메라 연출**: 순수 2D 카메라보다 3D 카메라(Perspective)의 자유도가 높습니다. 심문 중 긴장된 순간의 **줌인(Zoom-in), 틸트(Tilt), 쉐이크(Shake)** 연출 시 입체적인 화면 왜곡을 통해 극적인 이미지를 전달합니다.

## 🎨 [부록 2] 2D 파이프라인(URP 2D Renderer)의 작업적 장점

본 프로젝트에서 3D 공간을 활용하되, **URP 2D Renderer**를 병행 또는 대안으로 검토할 때 얻을 수 있는 실무적 이점은 다음과 같습니다.

### **1. 레이어 관리의 직관성 (Sorting Layers)**
- **정교한 순서 제어**: Z축의 물리적 거리 조절 없이 **Sorting Layer**와 **Order in Layer** 시스템을 통해 오브젝트의 렌더링 순서를 완벽하게 제어할 수 있습니다.
- **Z-Fighting 방지**: 스프라이트 간의 미세한 간격 차이로 인해 발생하는 깜빡임(Z-Fighting) 현상을 근본적으로 차단합니다.

### **2. 2D 전용 도구(Tilemap, Sprite Mask)의 활용**
- **타일맵(Tilemap)**: 배경 제작 시 블록 쌓기 방식의 타일맵을 활용하여 스테이지 제작 속도를 극대화할 수 있습니다.
- **스프라이트 마스크(Sprite Mask)**: 마스킹 처리가 필요한 UI나 오브젝트 연출 시, 별도의 쉐이더 연산 없이 2D 전용 기능을 사용하여 가볍게 구현 가능합니다.

### **3. 워크플로우의 단순화 (Simplicity)**
- **리소스 직결**: 디자이너의 결과물을 별도의 평면(Quad) 생성이나 메시(Mesh) 변환 없이 즉시 `SpriteRenderer`에 적용할 수 있어, 리소스 임포트 시간이 단축됩니다.
- **러닝 커브**: 엔진의 2D 전용 기능을 그대로 활용하므로, 팀 내 개발 및 기획 파트의 기술적 진입 장벽이 낮습니다.

### **💡 전략적 제언: URP 2D Renderer의 채택**
3D의 **퍼스펙티브 연출**과 2D의 **작업 편의성** 사이에서 고민할 경우, **URP 2D Renderer**를 기반으로 하되, 캐릭터 연출 시에만 **3D 조명과 lilToon의 특수 연산**을 혼합하는 하이브리드 방식이 가장 효율적일 수 있습니다.

## 📝 [부록] lilToon 쉐이더 채택 이유 및 가치

### **1. lilToon이란?**
- **정의**: 유니티에서 가장 널리 사용되는 고성능 **툰(Toon/Cel) 쉐이더** 중 하나로, 3D 모델이나 평면 오브젝트를 고퀄리티 애니메이션 스타일로 렌더링하는 데 특화되어 있습니다.
- **특징**: 매우 가벼우면서도 외곽선(Outline), 림 라이트(Rim Light), 발광(Emission) 등 일본식 애니메이션 특유의 비주얼을 완벽하게 구현합니다.

### **2. 프로젝트 도입 가치 (필요성)**
- **비주얼 독보성**: 단순한 2D 스프라이트를 넘어, **'움직이는 고퀄리티 애니메이션'**과 같은 비주얼을 선사하여 유저의 시각적 만족도를 극대화합니다.
- **심리적 상태의 시각화**: lilToon의 마스크(Mask) 기능을 활용하여, 심문 중 캐릭터가 당황할 때 나타나는 **눈동자의 흔들림, 패턴 레이어(식은땀 등) 등의 이펙트**를 물리적 리소스 제작 없이 쉐이더 연산만으로 자연스럽게 표현 가능합니다.
- **광원 동기화**: 마왕성 내부의 어두운 조명과 캐릭터의 밝은 애니메이션 톤 사이의 이질감을 줄여주며, 주변 환경 광원에 맞춘 실시간 셀 채색(Cel-shading)을 지원하여 공간의 통일감을 부여합니다.

### **3. 단계별 도입 계획**
- **Prototype**: 기본적인 URP 2D Lighting 및 기본 쉐이더로 시스템 검증 선행.
- **Alpha/Production**: 캐릭터 일러스트가 확정되는 시점에 lilToon 쉐이더를 적용하여 **최종 비주얼 폴리싱(Polishing)** 작업 수행.
